\section{Master manifest}

\begin{center}
\path{data/output/validation_reboot/validation_total_master_manifest.json}
\end{center}

with the following top-level structure:

\begin{itemize}
  \item \texttt{summary} -- ctest counters and timing.
  \item \texttt{phases} -- per-phase status, audit JSON pointers, and
        the relevant drivers/ctests/manifests.
  \item \texttt{cross\_formulation\_chain\_evidence} -- the eight
        kinematic\(\,\times\,\)amplitude campaigns and their stage
        outcomes.
  \item \texttt{honest\_scientific\_status} -- explicit segregation
        of (i) real runtime evidence, (ii) synthetic-chain plumbing
        evidence, and (iii) scoped-deferred items.
\end{itemize}

This manifest is the canonical entry point for any external auditor
or merge reviewer.

\section{Reproducibility}

The full validation can be reproduced from a clean checkout of
branch \texttt{multiscale-v2} via:

\begin{verbatim}
cmake --preset ninja-release
cmake --build build
ctest --test-dir build -L unit -j4
ctest --test-dir build -L validation_reboot -j4 \
   -E "cyclic_continuation_sensitivity|cyclic_node_refinement|\
       cyclic_quadrature_sensitivity|moment_curvature_closure_matrix"
\end{verbatim}

The cross-formulation chain log is regenerated by replaying any of
the eight \path{data/output/cyclic_validation/xfem_*_v2/hysteresis.csv}
files through the four standalone drivers with \texttt{--num-sites 3}.
