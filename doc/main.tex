\begin{abstract}
This document provides a comprehensive technical reference for the
\texttt{fall\_n} nonlinear finite element library.  It covers the
implementation of simplex (tetrahedral) finite elements, numerical integration
strategies on simplicial domains, the engineering decisions made to ensure
robust post-processing, and the layered element architecture with its
policy-based design patterns.

Key topics include:
\begin{itemize}[nosep]
  \item Lagrange shape functions on the reference simplex and their isoparametric
        mapping to physical space;
  \item Classical simplex quadrature rules and their negative-weight pathology;
  \item The Stroud conical product rule with Gauss--Jacobi nodes via the
        Golub--Welsch algorithm as an all-positive-weight alternative;
  \item Diagnosis and resolution of the checkerboard artefact in lumped $L^2$
        projections on quadratic tetrahedra;
  \item Polynomial patch recovery as the production nodal reconstruction for
        higher-order simplex visualisation, with volume-weighted averaging kept
        as a conservative fallback;
  \item An adaptive material-field projection strategy that distinguishes
        between element families with strictly positive local lumping and
        those that require robust local reconstruction;
  \item The refactor of \texttt{ElementGeometry}: corrected value semantics,
        explicit quadrature strategy contracts, compile-time invariant
        enforcement, and preservation of runtime performance in the hot path;
  \item Serendipity tensor-product and simplex element families, including
        static reference cells, benchmark validation, VTK support, and Gmsh
        import paths for HEX20 and QUA8;
  \item A detailed ontology for the \texttt{Domain} refactor, separating
        geometry, mesh vertices, nodal unknown carriers, integration sites,
        constitutive points, and structural section entities under a
        compile-time-oriented HPC design philosophy;
  \item The first implementation phase of the \texttt{Domain} refactor:
        canonical \texttt{Vertex} storage, lazy PETSc mesh creation,
        transitional nodal caching, and geometry-only mesh export through
        vertices;
  \item The second implementation phase of the \texttt{Domain} refactor:
        geometry-only element binding through \texttt{Point}/\texttt{Vertex},
        optional analysis-node binding, vertex-driven boundary extraction, and
        preservation of the compile-time/HPC characteristics of the element
        kernels;
  \item A new structural-scale reconstruction layer for beams and shells:
        constitutive section snapshots, formulation-specific reduction
        policies, compile-time section profiles, and a VTK multi-block export
        architecture that separates axis, surface, sections, material sites,
        and fibers;
  \item A quadrature-postprocessing clarification for VTK/ParaView:
        explicit nodal and quadrature tensor naming, scalar component export,
        and invariant filters such as von Mises, Beltrami--Haigh, pressure,
        triaxiality, and equivalent strain to avoid ambiguous rendering of
        raw Voigt arrays in quadrature-aware pipelines;
  \item A low-level performance pass on point, vertex, node, and element
        geometry access paths: contiguous coordinate access, direct DoF pointer
        access, cached coordinate pointers inside geometric elements, and
        removal of residual geometry-via-node usage in beam frame
        construction;
  \item A spatial-site and \texttt{Domain} laziness pass: a read-only
        geometry-site concept for point-like entities, explicit forwarding for
        constitutive and section decorators, and geometry-first boundary
        extraction without eager node-cache materialisation;
  \item A semantic hardening pass on \texttt{DMPlex}: mixed 1D/2D/3D cell
        families can now coexist in the same non-interpolated DAG through
        explicit point-role, topological-dimension, and physical-group labels,
        while the algebraic layout is driven by depth-$0$ vertices rather than
        brittle height-based assumptions;
  \item A full mixed-building demonstration in the executable entry point:
        first an elastic baseline, and then a materially nonlinear
        reinforced-concrete continuation with fiber sections in the frame,
        elastic MITC4 slabs, incremental SNES solution, and multi-block
        structural VTK export including fiber-level fields;
  \item A dedicated architectural audit of the materials module: semantic
        separation between physical materials, constitutive laws, integration
        algorithms, constitutive state carriers, and structural reduction
        policies, together with a roadmap toward nonlinear reinforced
        concrete across scales, finite kinematics, dynamics, and
        continuum-scale cracking/fracture;
  \item A constitutive-state storage cleanup: explicit committed-only,
        history-tracking, and trial/committed storage policies, with the
        default inelastic path moved away from implicit full kinematic
        history and toward semantically correct committed-state storage;
  \item A new semantic \texttt{ConstitutiveState} layer above the storage
        adapter: the local state of a constitutive site is now explicit in the
        API, while static polymorphism and configurable storage backends are
        preserved;
  \item An explicit \texttt{ConstitutiveIntegrator} layer at the erased
        \texttt{Material<Policy>} boundary: the local constitutive algorithm is
        now a first-class injection point, with legacy \texttt{ElasticUpdate}
        and \texttt{InelasticUpdate} retained as compatibility aliases;
  \item A first behavioural extraction of irreversible memory out of the law:
        \texttt{PlasticityRelation} now supports an explicit external
        algorithmic-state path, and \texttt{MaterialInstance} stores that state
        at the constitutive-site level while preserving the legacy embedded
        API for compatibility;
  \item The propagation of that explicit algorithmic-state contract to
        hysteretic uniaxial steel and concrete laws: \texttt{MenegottoPintoSteel}
        and \texttt{KentParkConcrete} now support external state objects, and
        fiber sections inherit the new behaviour automatically through the
        existing type-erased uniaxial-material layer;
  \item An explicit compile-time \texttt{YieldFunction} customization point in        plasticity, separated from yield-surface geometry and hardening, plus
        a constitutive protocol sampler and VTK curve writer to generate
        strain--stress hysteresis loops for direct visual inspection of
        nonlinear materials;
  \item A further split between constitutive ingredients and the local
        nonlinear solve: \texttt{PlasticityRelation} now exposes a
        compile-time \texttt{ReturnAlgorithm} axis, so the scalar yield
        function and the return-mapping strategy can evolve independently;
  \item A further split inside the local plastic solve itself: the scalar
        consistency residual and its Jacobian are now independent compile-time
        customization points, so future constitutive models can keep a Newton
        solver while changing the local equation, or keep the local equation
        while changing the nonlinear solver;
  \item A general local constitutive-problem framework above that scalar path:
        local updates are now expressible as compile-time problems
        $R(x)=0$ with reusable Newton solvers, making the current scalar
        consistency equation only a special case and preparing the route to
        finite-strain plasticity, damage, and other vector-valued local
        updates;
  \item A further internal split of Newton itself: the local linear solve and
        the step-control policy are now independent compile-time injection
        points, so Newton is no longer treated as a monolithic algorithm and
        future PETSc-based or research-oriented local solvers can attach to the
        same constitutive local-problem contract;
  \item A continuum-side constitutive carrier for large displacements and
        finite strains: the \texttt{ContinuumElement} /\texttt{Material<>}
        boundary no longer assumes that every continuum law sees only a flat
        Voigt strain vector, while remaining backward-compatible with the
        existing constitutive API, and a spatial updated-Lagrangian route now
        exposes Almansi strain and Cauchy stress consistently through the same
        erased constitutive boundary;
  \item A bug fix in PETSc~3.24.5 \texttt{section.c} and application-level
        workarounds for PETSc indexing and preallocation issues;
  \item The element architecture design: External Polymorphism at the geometry
        and assembly levels, policy-based customisation points
        (MaterialPolicy, KinematicPolicy, ElementPolicy), compile-time
        efficiency, and comparison with OpenSees and Kratos Multiphysics;
  \item A complete step-by-step guide to building and running \texttt{fall\_n}
        on Windows using the MSYS2 UCRT64 toolchain (GCC~15, CMake, Ninja)
        with all dependencies installed from the \texttt{pacman} binary
        repository, together with a detailed record of every platform-specific
        source and build-system change required.
\end{itemize}
Each chapter includes the relevant mathematical theory with bibliographic
references, the concrete C++ implementation in \texttt{fall\_n}, and a
discussion of the design rationale.
\end{abstract}
