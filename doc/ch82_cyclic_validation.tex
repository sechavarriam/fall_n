\chapter{Progressive Cyclic Validation of the FE² Pipeline}
\label{ch:cyclic-validation}

\paragraph{Normative status.}
This chapter remains an application and validation record.  The authoritative
definition of the current coupling modes and public multiscale contracts is
Chapter~\ref{ch:multiscale-publication-audit}.

This chapter documents a progressive validation suite for the
nonlinear analysis and FE² multiscale pipeline.  Seven cases of
increasing complexity are solved under a common cyclic displacement
protocol with geometric amplitude progression, using a ``4-legged
table'' geometry as the base structure.  The suite serves three
purposes: (i)~validating each modelling layer in isolation before
coupling, (ii)~quantifying the convergence behaviour of the
Ko--Bathe concrete model under cyclic loading, and
(iii)~establishing a reproducible baseline for the FE² two-way
coupling with an explicitly audited material-tangent strategy.

Beyond the numerical results, the chapter documents three aspects
discovered during the initial validation campaign: implementation
bugs in the sub-model solver
(Section~\ref{sec:cyclic-bugs}), the root-cause analysis of the
two-way coupling failure (Section~\ref{sec:case5-failure}), and a
detailed cross-reference between the Ko--Bathe article and the
\texttt{KoBatheConcrete3D} implementation, including the derivation
and implementation of the experimental numerically consistent tangent
(Sections~\ref{sec:article-crossref}--\ref{sec:consistent-tangent}).

\section{Motivation}
\label{sec:cyclic-motivation}

The seismic table analysis of Chapter~\ref{ch:table-multiscale}
demonstrated the full FE² pipeline but posed challenges in isolating
the source of convergence difficulties.  This validation suite
replaces the dynamic seismic excitation with a quasi-static
displacement-controlled cyclic protocol, guaranteeing that each
amplitude level is reached regardless of dynamic effects.  The
progressive structure isolates complexity layer by layer:

\begin{itemize}
  \item \textbf{Case~0:} Linear elastic reference beam,
    providing an analytical baseline.
  \item \textbf{Case~1:} Single beam element with RC fiber sections
    (Timoshenko order $N = 2\ldots10$), validating the section-level
    constitutive response and its sensitivity to integration order.
  \item \textbf{Case~2:} Single continuum column (Hex8/Hex20/Hex27)
    with Ko--Bathe concrete, validating the 3D constitutive model
    under cyclic loading in a displacement-controlled setting.
  \item \textbf{Case~3:} Full table structural model (4 beams +
    MITC16 shell), validating the mixed beam--shell nonlinear
    assembly under the complete protocol.
  \item \textbf{Case~4:} Table with FE² sub-models, one-way
    coupling (macro $\to$ micro only).
  \item \textbf{Case~5:} Table with FE² sub-models, two-way
    staggered coupling (macro $\leftrightarrow$ micro), using the
    fracture-sec tangent by default and retaining the consistent tangent
    only as an explicit audit mode.
\end{itemize}

\noindent
Cases~0--3 are fully operational.  Cases~4--5 reuse the table
geometry and protocol from Cases~3 but add the multiscale sub-model
layer.  A future Case~6 (single-element multiscale bridge, $N=5$
beam $\leftrightarrow$ 4 continuum sub-models) is planned to
validate the kinematics extraction at intermediate Gauss-point
sections.

\section{Cyclic Displacement Protocol}
\label{sec:cyclic-protocol}

\subsection{Design Rationale}

The original v1~protocol used amplitude levels expressed as multiples
of a nominal yield displacement: $\delta_k \in
\{1,2,4,8\}\times\delta_y$ with $\delta_y \approx 0.01$\,m.  This
proved unsuitable because (i)~the yield displacement depends on the
material model and element formulation, making cross-case comparison
ambiguous, and (ii)~the maximum amplitude of $80$\,mm was excessive,
driving most nonlinear cases into deep softening where the secant
tangent becomes nearly singular.

The validation driver now exposes two explicit protocol presets based on
\emph{absolute} geometric amplitudes:
\begin{align}
  A_k^{\text{legacy20}} &\in \{2.5,\; 5.0,\; 10.0,\; 20.0\}\;\text{mm}, \\
  A_k^{\text{extended50}} &\in \{2.5,\; 5.0,\; 10.0,\; 20.0,\; 35.0,\; 50.0\}\;\text{mm}.
  \label{eq:protocol-amplitudes}
\end{align}
The \texttt{legacy20} set is preserved for backward comparison with the earlier
campaign.  The current validation default is \texttt{extended50}, because it
drives the table cases far enough to expose crack initiation, large hysteretic
loops, and the current multiscale runtime limitations.  For the present table
geometry ($H = 3.2$\,m), the maximum drift ratio of the extended protocol is
\(50/3200 = 1.56\%\).

\subsection{Protocol Function}

Each amplitude level consists of three linear segments:
$0 \to +A_k$, $+A_k \to -A_k$, and $-A_k \to 0$, giving
\(N_{\text{seg}} = 3\,n_A\) segments, where \(n_A\) is the number of amplitude
levels in the selected preset.  The driver also exposes a configurable number
of solver steps per segment.  Therefore the total number of macro steps is
\begin{equation}
  N_s = 3\,n_A\,n_{\text{seg-step}},
\end{equation}
with \(n_{\text{seg-step}} = 10\) in the full-resolution default runs and
\(n_{\text{seg-step}} = 1\) in the coarse exploratory FE\textsuperscript{2}
runs used during the present stabilization cycle.

The displacement function is implemented in
\texttt{src/utils/CyclicProtocol.hh} as:
\begin{equation}
  d(p) = \begin{cases}
    f \cdot A_k        & \text{phase 0 (ascending: } 0 \to +A_k\text{)} \\
    A_k(1 - 2f)        & \text{phase 1 (descending: } +A_k \to -A_k\text{)} \\
    -A_k(1 - f)        & \text{phase 2 (return: } -A_k \to 0\text{)}
  \end{cases}
  \label{eq:protocol-function}
\end{equation}
where $A_k$ is the current amplitude from~\eqref{eq:protocol-amplitudes},
$f = t - \lfloor t \rfloor$ is the fractional position within
the current segment, $t = p \times N_{\text{seg}}$, the amplitude
level is $k = \lfloor t \rfloor / 3$, and the phase is
$\lfloor t \rfloor \bmod 3$.

  \caption{Cyclic displacement protocol generated by
    \texttt{src/utils/CyclicProtocol.hh}.  The current validation default is
    \texttt{extended50}, with amplitude levels
    $\{2.5, 5, 10, 20, 35, 50\}$\,mm and a configurable number of solver steps

\subsection{Unit Tests}

The protocol contract is guarded by
\texttt{tests/test\_cyclic\_protocol.cpp}.  The current test matrix validates
both presets (\texttt{legacy20} and \texttt{extended50}) against the same
properties:
\begin{enumerate}
  \item \textbf{Endpoints:} $d(0) = 0$, $d(1) = 0$.
  \item \textbf{Peak amplitudes:} both presets hit the prescribed positive and
    negative drift levels exactly at their segment endpoints, including the new
    $\pm 35$\,mm and $\pm 50$\,mm peaks.
  \item \textbf{Cycle symmetry:} each level returns to zero within
    tolerance.
  \item \textbf{Known values:} spot-checks at mid-segment positions.
  \item \textbf{Monotonicity:} ascending phase is strictly increasing,
    descending phase is strictly decreasing.
\end{enumerate}

\paragraph{Current stabilized status.}
The present validation driver uses the \texttt{extended50} preset by default,
records crack and solver-diagnostic CSVs, and forces the
\texttt{SerialExecutor} for the cyclic FE\textsuperscript{2} path because an
\texttt{OpenMPExecutor} attempt exposed a PETSc wrong-state error
(\texttt{VecNormBegin/End} ordering mismatch).  The most informative runs of
the current cycle were the truncated one-way FE\textsuperscript{2} campaigns
of Case~4 under the \texttt{extended50} protocol.  Two observations are now
normative:
\begin{enumerate}
  \item the sub-model constitutive law \texttt{KoBatheConcrete3D} does
    \emph{not} expose a scalar damage channel through the public
    \texttt{InternalFieldSnapshot} path, so the cyclic recorder must report
    \texttt{damage\_scalar\_available = 0} and write \texttt{NaN} instead of
    fabricating ``zero damage'';
  \item the physically meaningful crack indicators for the present cyclic FE\textsuperscript{2}
    path are therefore the cracked Gauss-point count, total crack planes,
    maximum crack opening, and the octahedral fracture-history envelopes
    \(\sigma_{o,\min}\) and \(\tau_{o,\max}\).
\end{enumerate}

The driver now also exposes an explicit execution profile,
\texttt{fe2\_crack50}, intended for long exploratory
FE\textsuperscript{2} runs under \texttt{extended50}.  That profile applies
one macro step per protocol segment, a 12-substep first ramp, adaptive local
continuation from the first post-ramp reversal, enlarged adaptive budgets
\((48,12)\), increased first-ramp and SNES budgets, disabled
structural/sub-model VTK output, and
\texttt{min\_crack\_opening}=0 so that crack summaries remain sensitive even
when graphical output is suppressed.  The intent is not to redefine the
physics, but to remove avoidable I/O and configuration noise while probing the
current runtime envelope of the multiscale path.

The first positive point of the protocol, i.e.\ the step to \(+2.5\)\,mm
drift, was reproduced with a reduced 8-substep \texttt{FirstSolveRamp} instead
of the older 30-substep baseline.  That run required approximately 439~s in
the audited environment and the resulting recorders reported:
\begin{itemize}
  \item macro base shear \(V = -0.5678\)~MN;
  \item structural peak fiber damage \(= 0.0217028\);
  \item 360 cracked Gauss points and 1080 total cracks across the four
    sub-models;
  \item peak crack opening \(= 9.88515\times 10^{-4}\);
  \item \(\sigma_{o,\min} = -0.990751\) and \(\tau_{o,\max} = 8.60697\);
  \item four local responses with \texttt{ResponseStatus::Ok},
    \texttt{TangentLinearizationScheme::LinearizedCondensation}, and
    \texttt{CondensedTangentStatus::Success}.
\end{itemize}

A second exploratory run enabled adaptive sub-stepping from the start of the
post-ramp phase.  With the same 8-substep first ramp, the driver completed the
first four protocol points
\(\{+2.5\text{ mm}, -2.5\text{ mm}, 0, +5.0\text{ mm}\}\) in
approximately 4870~s.  At \(+5.0\)\,mm drift the one-way
FE\textsuperscript{2} recorders reported:
\begin{itemize}
  \item macro base shear \(V = -1.13555\)~MN;
  \item structural peak fiber damage \(= 0.0434864\);
  \item 426 cracked Gauss points and 1268 total cracks;
  \item peak crack opening \(= 1.82279\times 10^{-3}\);
  \item \(\sigma_{o,\min} = -2.23149\) and \(\tau_{o,\max} = 12.2076\);
  \item four accepted local responses with
    \texttt{SubsequentAdaptiveStep} and successful condensed tangents.
\end{itemize}

This evidence changes the diagnosis of Case~4 substantially.  The current
bottleneck is neither immediate operator failure nor missing cracking data.
It is the combination of:
\begin{itemize}
  \item high cost per post-cracking step,
  \item the need for adaptive local continuation already at the first load
    reversal, and
  \item the growing gap between the structural-only damage envelope and the
    continuum crack/fracture indicators.
\end{itemize}

An additional stabilization pass then targeted long exploratory runs beyond the
\(+5\)\,mm point.  Two implementation changes are relevant:
\begin{enumerate}
  \item when sub-model VTK is disabled, the evolver now computes only a cached
    \texttt{CrackSummary} instead of rebuilding the full \texttt{CrackRecord}
    geometry required for crack-plane VTK export;
  \item the FE\textsuperscript{2} \texttt{hysteresis.csv} file is now streamed
    incrementally, step by step, instead of being written only at the end of
    the run.
\end{enumerate}
These changes matter because the long cyclic FE\textsuperscript{2} campaigns
are now interruption-safe: even when a run is truncated by an external timeout,
the global hysteresis, crack, and solver-diagnostic recorders remain
scientifically usable.

Under that updated implementation, a new one-way Case~4 attempt with
\texttt{--protocol extended50 --fe2-profile crack50 --max-steps 5} was allowed
to run for two hours.  The run still did not reach the \(-5\)\,mm reversal
within the available wall-clock budget, but the streamed recorders confirmed
that the stabilized FE\textsuperscript{2} path again completed the first four
protocol points through \(+5\)\,mm, with no loss of hysteresis or crack data.
Therefore the current evidence frontier is now explicit:
\begin{itemize}
  \item \textbf{Case~3} already completes the full \texttt{extended50} protocol
    to \(50\)\,mm and currently provides the clearest full-amplitude
    hysteretic reference of the library;
  \item \textbf{Case~4} one-way FE\textsuperscript{2} is scientifically
    observable through \(+5\)\,mm with crack-history metrics and reliable
    partial-run recorders, but it remains runtime-limited beyond that point in
    the audited environment.
\end{itemize}

To make that frontier executable rather than merely narrative, the repository
now contains the script
\texttt{scripts/run\_predefinitive\_physical\_validation.ps1}.  The script
reruns the focused regression surface, executes one short one-way
FE\textsuperscript{2} campaign (Case~4, one protocol point), executes one
short iterated two-way FE\textsuperscript{2} campaign (Case~5, one protocol
point), stores the raw logs, and writes a machine-readable summary CSV.  The
summary vocabulary is intentionally explicit:
\begin{itemize}
  \item \texttt{TRUNCATED\_OK} means that the run was intentionally capped but
    still produced scientifically usable recorders;
  \item \texttt{KNOWN\_FRONTIER} means that the run stopped at the currently
    documented failure frontier with the expected rollback and failed-site
    semantics.
\end{itemize}
On the audited short-run matrix, Case~4 produced
\(d = 2.5\times 10^{-3}\)~m,
\(V=-0.5678\)~MN, 1080 cracks, and
\(9.88515\times 10^{-4}\) peak opening in 342.371~s.  That initial matrix also
captured an iterated two-way Case~5 failure at the same \(+2.5\)\,mm point.
However, the subsequent stabilization pass corrected the Ko--Bathe factory
parameter mapping, aligned the inferred local crack-band length with the
max-side rule of Fig.~7(b) in~\cite{KoBathe2026}, and upgraded local
checkpoint/restore to preserve the full constitutive state during trial
iterations.  After those corrections, the same Case~5 first cracked point
converged under the tuned \texttt{fe2\_crack50} budget with:
\begin{itemize}
  \item \(p = 0.0555556\), corresponding to \(+2.5\)\,mm drift;
  \item base shear \(V = 1.50812\)~MN;
  \item peak structural damage \(= 0.0214922\);
  \item 296 cracked Gauss points and 880 crack planes;
  \item peak crack opening \(= 9.84163\times 10^{-4}\);
  \item zero failed sub-models and two outer FE\textsuperscript{2} iterations.
\end{itemize}
This does not close the full physical validation campaign, but it moves the
frontier materially: the first cracked point of iterated two-way
FE\textsuperscript{2} is now crossed reproducibly, and the open frontier shifts
to later amplitudes of the \texttt{extended50} protocol.

\paragraph{Historical v2 record.}

\subsection{Outputs and Recorders}

The cyclic validation driver now writes more than a single
\texttt{hysteresis.csv}.  In addition to the global base-shear history, the
structural and FE\textsuperscript{2} table cases generate:
\begin{itemize}
  \item \texttt{recorders/global\_history.csv}: step-wise drift, base shear, and
    structural damage, plus the availability-flagged sub-model fracture
    indicators;
  \item \texttt{recorders/fiber\_hysteresis\_*.csv}: concrete and steel fiber
    stress--strain loops;
  \item \texttt{recorders/crack\_evolution.csv}: FE\textsuperscript{2} crack
    counts, cracked Gauss points, peak opening, and octahedral
    fracture-history envelopes;
  \item \texttt{recorders/solver\_diagnostics.csv}: per-step coupling
    termination reason, failed sites, local response status, tangent scheme,
    and local SNES diagnostics for each sub-model.
\end{itemize}

The companion plotting script
\texttt{scripts/plot\_cyclic\_validation.py} was extended accordingly; when the
CSV files are present, it now produces not only the global hysteresis overlays
but also per-case plots for fiber hysteresis, crack evolution, global damage
history, octahedral fracture-history envelopes, and solver diagnostics.
In addition, the script now supports a dependency-light
\texttt{--summary-only} mode, which prints per-case textual summaries without
requiring \texttt{numpy}, \texttt{pandas}, or \texttt{matplotlib}.  This mode
is particularly useful while auditing long FE\textsuperscript{2} runs on
environments that can execute the solver but lack the full plotting stack.

\section{Case 0: Linear Elastic Reference}
\label{sec:cyclic-case0}

A single vertical column from $z = 0$ (clamped) to $z = H$
(displacement-controlled in~$X$) is modelled with
\texttt{TimoshenkoBeamN<3>} (3-node quadratic beam, 2~Gauss points)
using \texttt{TimoshenkoBeamMaterial3D} with
\texttt{ElasticUpdate}---i.e.\ a purely elastic material with
no fiber section.

The elastic section properties are computed analytically:
\begin{equation}
  E_c = 4700\sqrt{f'_c},\quad
  G_c = \frac{E_c}{2(1+\nu)},\quad
  I_y = I_z = \frac{bh^3}{12},\quad
  J = 0.1406\,bh^3,\quad
  k_s = \tfrac{5}{6}
  \label{eq:case0-props}
\end{equation}
with $b = h = 0.25$\,m, $\nu = 0.2$.  This produces an elastic
cantilever stiffness of $k_{\text{el}} = 3EI/H^3 = 0.764$~MN/m.

\paragraph{Earlier v2 results.}
Case~0 completed all 120 steps in 1~Newton iteration per step
(as expected for a linear problem).  The base-shear--drift
response is perfectly linear and proportional:

\begin{center}
\begin{tabular}{rcc}
\toprule
\textbf{Step} & $\delta$~(mm) & $V$~(MN) \\
\midrule
 20 & $-2.5$  & $+1.910 \times 10^{-3}$ \\
 40 & $+5.0$  & $-3.819 \times 10^{-3}$ \\
 60 & $0.0$   & $\approx 0$ \\
 80 & $-10.0$ & $+7.638 \times 10^{-3}$ \\
100 & $+20.0$ & $-1.528 \times 10^{-2}$ \\
120 & $0.0$   & $\approx 0$ \\
\bottomrule
\end{tabular}
\end{center}

\noindent
The ratio $|V/\delta| = 0.764$~MN/m is constant at all peaks,
confirming the analytical Timoshenko cantilever stiffness.  This
case serves as the elastic reference envelope for all subsequent
comparisons.

\paragraph{Discussion.}
Several trends emerge from Table~\ref{tab:case1-results}:

\begin{enumerate}
  \item The effective secant stiffness degrades with increasing
    amplitude.  For Case~1a ($N=2$), the stiffness ratio
    $|V/\delta|$ drops from $1.42$~MN/m at $\pm 2.5$\,mm to
    $0.48$~MN/m at $\pm 20$\,mm (a $66\%$ reduction), consistent
    with concrete cracking and steel yielding progressively degrading
    the section.

  \item Higher-order elements fail earlier because more Gauss points
    distribute the sectional integration along the element length.
    With a \emph{single} element, the plastic hinge must localise at
    a single integration point; the $N=2$ element (1~GP) concentrates
    all inelastic strain there, while $N=5$ (4~GPs) spreads the demand
    and triggers the secant tangent singularity at multiple locations
    simultaneously.

  \item Cases~1a and~1b both complete the full protocol, demonstrating
    that the fiber-section constitutive response, Newton solver, and
    bisecting incremental framework are robust for low-order beams.

  \item The nonlinear base shear exceeds the elastic reference at
    small amplitudes ($3.55 > 1.91$\,kN at $2.5$\,mm) because the
    initial tangent stiffness of the RC section is higher than the
    Timoshenko elastic stiffness---the fiber section includes steel
    reinforcement, which stiffens the cross-section beyond plain
    concrete.
\end{enumerate}

\section{Case 2: Single Reinforced Continuum Column}
\label{sec:cyclic-case2}

A prismatic column ($0.25 \times 0.25 \times 3.2$~m) is meshed with
hexahedral concrete elements using a $2 \times 2 \times 12$ grid,
reinforced with 8~longitudinal rebar truss elements coupled via the
penalty formulation described in Section~\ref{sec:cyclic-penalty-coupling}.
The concrete material is \texttt{KoBatheConcrete3D} ($f'_c = 30$\,MPa)
and the steel is \texttt{MenegottoPintoSteel} ($E = 200$\,GPa,
$f_y = 420$\,MPa, $b = 0.01$).  The penalty parameter is
$\alpha = 10\,E_c \approx 2.6 \times 10^5$\,MN/m.

\begin{itemize}
  \item \textbf{Case~2a:} Hex8 (trilinear) --- 48~hex +
    96~truss elements, 221~nodes.  All 120~steps converged
    (121~records), completing the full four-cycle protocol
    ($\pm 2.5, 5, 10, 20$\,mm).
  \item \textbf{Case~2b:} Hex20 (serendipity) --- 48~hex +
    96~truss elements, 825~nodes.  101~records (steps~0--100),
    completed the first three cycles and the push leg of the 4th
    ($+20$\,mm peak), then aborted at the load reversal when
    bisection was exhausted.
  \item \textbf{Case~2c:} Hex27 (triquadratic) --- 48~hex +
    96~truss elements, 825~nodes.  97~records (steps~0--96),
    completed the first three cycles and reached $\delta = +12$\,mm
    during the 4th-cycle push before bisection was exhausted.
\end{itemize}

\section{Case 4: Table with FE² Sub-Models (One-Way Coupling)}
\label{sec:cyclic-case4}

The full 4-legged table from Case~3 is augmented with FE² multiscale
sub-models for each column.  The macro-scale model uses
\texttt{BeamElement<TimoshenkoBeam3D, 3, beam::SmallRotation>}
columns and an MITC16 shell slab, identical to Case~3.  At each
global step, a \textbf{one-way} coupling procedure:

\begin{enumerate}
  \item Extracts section kinematics (curvatures and axial strain)
    at both endpoints of each beam column.
  \item Solves a 3D RC sub-model ($2 \times 2 \times 1$ Hex27 mesh
    with 8 embedded rebar bars) using
    \texttt{NonlinearSubModelEvolver}, which applies Ko--Bathe
    concrete and Menegotto--Pinto steel with penalty-coupled rebar
    ($\alpha = 10\,E_c$).
  \item Computes the section forces and tangent from the
    boundary-reaction operator, preferring the condensed linearization path
    whenever the local operator is available.
  \item Records the local response status, crack summary, and solver
    diagnostics for each sub-model.  Because this is \emph{one-way}
    downscaling, the accepted macro state of the current step still comes from
    the structural solve of the beam--shell model.
\end{enumerate}

\paragraph{Current stabilized status.}
The present stabilized FE\textsuperscript{2} driver no longer assumes a fixed
``60-step'' campaign.  Instead, the protocol, manual step cap, first-ramp
resolution, and adaptive post-ramp continuation are all explicit runtime
controls.  In the audited one-way runs the relevant local solver parameters
were:
\begin{itemize}
  \item \texttt{max\_it} $= 100$, \texttt{atol} $= 2.0$,
    \texttt{rtol} $= 10^{-3}$;
  \item initial \texttt{FirstSolveRamp} with 8 sub-steps and 3 bisection
    levels;
  \item \texttt{SerialExecutor} at the validation-driver level;
  \item adaptive sub-stepping enabled from the first post-ramp step.
\end{itemize}

\section{Case 5: Table with FE² Sub-Models (Two-Way Coupling)}
\label{sec:cyclic-case5}

Identical to Case~4 in geometry and loading, but the coupling
strategy is upgraded to \texttt{IteratedTwoWayFE2} with a maximum of
3 fixed-point iterations per global step, force-and-tangent
convergence with tolerance $\varepsilon_{\text{stag}} = 0.05$, and
constant relaxation $\omega = 0.8$.

\paragraph{Current stabilized status.}
The two-way cyclic FE\textsuperscript{2} case remains an open scientific and
runtime debt.  The driver now exposes the same crack, damage, and
solver-diagnostic recorders as Case~4, and the new manual FE\textsuperscript{2}
step cap preserves the protocol parametrization for exploratory runs.
However, the full \texttt{extended50} protocol has not yet been closed
end-to-end in the present cycle because even the early multiscale steps remain
prohibitively expensive.  The honest state of the project is therefore:
Case~5 is architecturally instrumented and better diagnosed than before, but it
is not yet a release-quality long-run validation case.

An important correction to the historical narrative became necessary in the
present pass.  After the cyclic FE\textsuperscript{2} setup was split into a
separate translation unit, the returned local evolvers still held raw
\texttt{MultiscaleSubModel*} pointers into a temporary
\texttt{MultiscaleCoordinator}.  Once the setup function returned, those
pointers became dangling, which in turn corrupted the cached face-node lists
used to rebuild the sub-model boundary conditions.  The immediate symptom was a
post-step abort inside the boundary-displacement reconstruction, not a genuine
material or FE\textsuperscript{2} convergence failure.  The corrected driver
now keeps the coordinator alive inside the returned case context and validates
the face-node caches before rebuilding the local boundary conditions.

\paragraph{Audited short-run benchmark.}
After the coordinator-lifetime correction, both Case~4 and Case~5 were
reprobed with the same short-run settings
\texttt{--protocol extended50 --fe2-profile crack50 --max-steps 1}.  Within the
same \(45\) to \(60\)\,s observation window, the two cases exhibited the same
macro state and entered the first local ramp without the previous abort:
\begin{itemize}
  \item the macro step converged at \(p = 0.055556\) in \(2\) Newton
    iterations;
  \item the first local solve reached at least the \(p=0.50\) ramp fraction in
    the audited probe;
  \item no post-step segmentation fault or corrupted face-node cache was
    observed;
  \item the limiting factor reverted to the true one: the wall-clock cost of
    the first cracked local ramp.
\end{itemize}
This means the active Case~5 frontier has moved again: before any definitive
statement is made about material convergence at the first cracked point, the
end-to-end short-run benchmark must be re-audited on top of the corrected
FE\textsuperscript{2} setup lifetime.

\paragraph{Constitutive branch correction in the current pass.}
The subsequent constitutive audit against Table~1 of
Ko--Bathe~\cite{KoBathe2026} showed that the 3D material was still allowing
the compression-coordinate return mapping to run implicitly in states that the
article classifies as \emph{no-flow}.  The implementation was corrected so
that only
\[
  {}^{t+\Delta t}\sigma_o^{\mathrm{tr}} < 0
  \quad\text{and}\quad
  {}^{t+\Delta t}\tau_o^{\mathrm{tr}} > {}^{t}\tau_o^{\max}
\]
can enter the plastic-flow branch; tensile states and compressive unloading
now remain explicitly on the no-flow branch, and the effective plastic strains
stay frozen there.  In the same pass, the article's Eq.~(26b) was also
implemented explicitly in the 3D no-flow branch: once a tensile or unloading
state is identified, the algorithm now recovers the final elastic strain from
the no-flow constitutive operator and updates the tensorial inelastic strain
through
\[
  {}^{t+\Delta t}\!\varepsilon^{p}_{ij}
  =
  {}^{t}\!\varepsilon^{p}_{ij}
  -
  {}^{t+\Delta t}\!\varepsilon^{e}_{ij}(N)
  +
  {}^{t+\Delta t}\!\varepsilon^{e}_{ij}(0).
\]
When the crack topology or open/closed status changes inside the same
increment, the no-flow branch now performs a bounded constitutive
stabilization loop before accepting the final state.  Hence the present
implementation no longer applies Eq.~(26b) after a single constitutive
restart; it iterates the cracked operator until the crack-status history
stabilizes, or until a small constitutive cap is reached.  This is still not a
literal transcription of the article's full \(m\)-history because the
crack-count reduction remains regularized, but it is materially closer to
Table~1 than the old one-shot approximation.

The short-run FE\textsuperscript{2} consequence is now best read in two
stages.  Historically, a very strict probe with
\texttt{--submodel-snes-max-it 20} appeared to fail before any observable
cracking could be diagnosed.  That result is now retained only as historical
evidence, because the validation recorders have since been hardened and the
named \texttt{fe2\_frontier\_audit} profile has made the same first cracked
point much more explicit.

Under the current frontier-audit interpretation, the first failed Case~5 point
still aborts as \texttt{MicroSolveFailed}, but it no longer looks like a
featureless pre-crack Newton collapse.  The failed-step recorder split now
shows that the open-crack observables remain zero at the failed target while
the constitutive crack-history observable is already large.  In the current
audited row,
\[
  \texttt{total\_cracked\_gps} = 0,
  \qquad
  \texttt{total\_active\_crack\_history\_points} = 306,
  \qquad
  \max_g n_c(g) = 3.
\]
Therefore the earliest failed cracked point is no longer ``no cracking'';
rather, it is a state in which crack history is already active in the material
while no open-crack observable survives the recorder threshold.

With the broader exploratory short-run probe
\texttt{--submodel-snes-max-it 60}, the same first macro step aborts only
later: Submodels~0 and~2 fail at \(k=3/4\), while Submodels~1 and~3 fail at
\(k=4/4\).  The recorded \texttt{solver\_diagnostics.csv} row then reports,
for every failed submodel, \(72\) active cracked material points, \(2\) to
\(3\) cracks at the most damaged points, one no-flow stabilization pass, zero
unstabilized no-flow states, and elastic-recovery residuals of order
\(10^{-15}\).  The scientific interpretation is therefore sharper than before:
the current Case~5 frontier is not well explained by a missing Eq.~(26b) term
or by an unstabilized \(m\)-loop surrogate.  The blocker has moved to the
Newton convergence of a heavily cracked local ramp in the range
\(p_{\mathrm{local}}\in[0.75,1.0]\).

\paragraph{Consistent-tangent audit result.}
The explicit
\texttt{--submodel-consistent-material-tangent} override remains available for
audit, but its interpretation also changed after the ownership fix.  The
historical one-step benchmark that ended in \texttt{MicroSolveFailed} is still
kept as evidence that the numerical tangent path is not automatically a
production win.  However, after the setup-lifetime correction, a fresh
\(45\)\,s probe with the same override again progressed into the first local
ramp rather than failing immediately and reached \(k=2/4\).  The newer
strict \texttt{max-it = 20} diagnostic probe also clarifies the failure mode:
the consistent tangent does \emph{not} change the local failure classification.
The four submodels still abort with \texttt{NewtonDiverged} at \(k=1/4\), and
the recorded residual norms are in fact larger than in the secant path.
Therefore the correct scientific statement is now more modest: the consistent
tangent is still experimental, but the current evidence does not support
promoting it as the immediate cure for the Case~5 frontier.

\paragraph{Root cause.}
The root cause of the previous immediate frontier was partly architectural, not
purely constitutive: a dangling sub-model ownership chain invalidated the local
boundary-condition rebuild immediately after the first macro step.  With that
bug removed, the remaining blocker is again the expensive nonlinear micro ramp
and the later reversals of the \texttt{extended50} protocol.  The
consistent-tangent audit remains informative, but it can no longer be used in
isolation to claim that the material tangent itself is the sole source of the
former first-point failure.

\paragraph{Implications.}
Two-way FE² coupling with the secant-based Ko--Bathe model requires
either (a)~a regularised homogenisation scheme that smooths the
tangent transitions, or (b)~a continuation/arc-length strategy at
the macro level.  Both approaches are beyond the current scope and
are flagged for future work.

Key observations from the comparison:

\begin{enumerate}
  \item \textbf{Elastic reference (Case~0):}  The linear response
    provides an upper bound for stiffness and a reference envelope.
    All nonlinear cases show smaller forces at large drifts, confirming
    that the nonlinear degradation is real and not a solver artifact.

  \item \textbf{Beam convergence (Case~1):} The linear element
    ($N = 2$, 1~GP) produces the widest hysteresis loop because the
    single integration point concentrates all inelastic strain.
    Higher orders ($N = 3, 4, 5$) distribute the integration and
    trigger softening at multiple points simultaneously, causing
    earlier divergence.  A multi-element mesh would be needed to
    properly resolve the plastic hinge with higher-order elements.

  \item \textbf{Continuum vs.\ beam (Cases 1a vs.\ 2a):}  The
    reinforced continuum model (Case~2a) achieves $k_{\text{eff}} =
    2.334$\,MN/m at 20\,mm, substantially stiffer than the beam
    model ($0.478$\,MN/m), reflecting the higher integration order
    and three-dimensional stress distribution.  Unlike the earlier
    unreinforced continuum run that aborted during the 3rd~cycle,
    the reinforced column completes all four cycles with only
    11.5\%~stiffness degradation, demonstrating that the penalty-coupled
    rebar provides the necessary tensile load path to sustain
    reversed loading through the Ko--Bathe cracking model.

  \item \textbf{Hex-order effect (Cases 2a vs.\ 2b vs.\ 2c):}
    The Hex20 serendipity model is $\sim$20\%~softer than Hex8
    ($k_1 = 2.089$ vs.~$2.636$\,MN/m) because the mid-edge nodes
    enable quadratic bending modes, reducing artificial stiffness
    from the coarse $2 \times 2 \times 12$~mesh.  The Hex27 model
    ($k_1 = 2.088$\,MN/m) is virtually identical to Hex20,
    confirming that the additional face-centre and body-centre
    nodes contribute negligible extra compliance.  Both higher-order
    models abort during the 4th cycle (Hex20 at step~100, Hex27 at
    step~96), whereas Hex8 completes all four cycles.  The
    additional internal DOFs distribute cracking more uniformly but
    create sharper tangent transitions during crack closure,
    exceeding the bisection algorithm's capacity at large amplitudes.

  \item \textbf{Table stiffness (Case~3):}  The table stiffness
    ($\sim 225$\,MN/m) is far larger than $4 \times$ the single-column
    stiffness ($4 \times 1.42 = 5.7$\,MN/m) because the MITC16 shell
    slab acts as a rigid diaphragm and contributes significant
    in-plane shear stiffness.  This is physically correct for a
    monolithic RC slab--column connection.

  \item \textbf{FE² one-way (Case~4):} Matches Case~3 at the
    $\pm 2.5$\,mm amplitude level, confirming that the sub-model
    solver, homogenisation, and injection procedure are consistent
    with the structural response when no feedback is applied.

  \item \textbf{Stiffness degradation pattern:} The ratio
    $k(20\,\text{mm}) / k(2.5\,\text{mm})$ varies by case:
    Case~0 = 1.00 (elastic), Case~1a = 0.34 (66\% loss),
    Case~1b = 0.35 (65\% loss), Case~2a = 0.885 (11.5\% loss),
    Case~2b = 0.923 (7.7\% loss at $+20$\,mm push peak),
    Case~2c $\approx$ 0.923 (virtually identical to 2b),
    Case~3 $\approx$ 0.99
    (single-GP beams localise all plastic strain, preserving
    global stiffness).  All three reinforced continuum models show
    mild degradation because the distributed rebar carries the
    tensile forces that would otherwise be lost to cracking.
    Hex20 and Hex27 are indistinguishable, confirming that
    the mid-edge shape functions dominate the quadratic response.
\end{enumerate}

\section{Phase~C: Penalty Coupling Architecture}
\label{sec:cyclic-penalty-coupling}

To enable embedded-rebar continuum analyses (Case~2 reinforced),
two new features were added to the solver infrastructure:

\paragraph{Residual/Jacobian hooks in \texttt{NonlinearAnalysis}.}
The SNES callbacks \texttt{FormResidual} and \texttt{FormJacobian}
now support user-registered hooks via
\texttt{set\_residual\_hook()} and \texttt{set\_jacobian\_hook()}.
Each hook is a \texttt{std::function} called after the standard
element assembly loop but before the local-to-global scatter
(residual) or \texttt{MatAssembly} (Jacobian).  This allows
coupling terms---such as penalty springs---to be injected without
modifying the element classes.

The hook signatures are:
\begin{itemize}
  \item \texttt{ResidualHook = std::function<void(Vec u\_local,
        Vec f\_local, DM dm)>}
  \item \texttt{JacobianHook = std::function<void(Vec u\_local,
        Mat J, DM dm)>}
\end{itemize}

\paragraph{Standalone \texttt{PenaltyCoupling} class.}
The file \texttt{src/analysis/PenaltyCoupling.hh} provides a
self-contained penalty coupling handler that enforces displacement
compatibility between rebar truss nodes and their host hexahedral
elements:
\begin{equation}
  \text{gap}_d = u^{\text{rebar}}_d
                 - \sum_{i} N_i(\xi,\eta,\zeta)\, u^{\text{hex}}_{i,d},
  \qquad
  \alpha_{\text{pen}} = 10\,E_c \approx 2.6 \times 10^{5}~\text{MPa}.
  \label{eq:penalty-gap}
\end{equation}
The residual contributions are:
\begin{equation}
  F^{\text{rebar}}_d \mathrel{+}= \alpha\, \text{gap}_d, \qquad
  F^{\text{hex}}_{i,d} \mathrel{-}= \alpha\, N_i\, \text{gap}_d,
\end{equation}
and the tangent stiffness additions are:
\begin{align}
  K_{rr}  &\mathrel{+}= \alpha\, I_3, &
  K_{rh_i} &\mathrel{+}= -\alpha\, N_i\, I_3, \\
  K_{h_i r} &\mathrel{+}= -\alpha\, N_i\, I_3, &
  K_{h_i h_j} &\mathrel{+}= \alpha\, N_i\, N_j\, I_3.
\end{align}
The class computes coupling entries in \texttt{setup()} by evaluating
tensor-product Lagrange shape functions at each rebar node's
parametric coordinates $(\xi,\eta,\zeta)$ within its host hex element.
Boundary-face rebar nodes (MinZ/MaxZ) are skipped since they are
handled via Dirichlet constraints.

\paragraph{Case~2 upgrade: reinforced column.}
The original Case~2 was an unreinforced hex-only column that aborted
at step~75 during the $\pm 10$\,mm cycle.  It has been replaced by a
reinforced column with 8~rebar bars (4~corner $+$ 4~mid-face), using
\texttt{MixedModel} (via \texttt{MultiElementPolicy} with
\texttt{FEM\_Element} type erasure) combining 48~hex $+$ 96~truss
elements.  Materials are \texttt{KoBatheConcrete3D} (consistent tangent)
$+$ \texttt{MenegottoPintoSteel} ($E_s=200$\,GPa, $f_y=420$\,MPa,
$b=0.01$).

\paragraph{Bug fix: singular Jacobian from unconstrained transverse DOFs.}
The initial implementation constrained only the $u_x$ DOF of
MaxZ-face rebar nodes (for cyclic displacement control), leaving
$u_y$ and $u_z$ free.  Since the penalty coupling skips boundary
faces, and truss elements only provide axial ($z$-direction)
stiffness, these two transverse DOFs had zero stiffness---making the
Jacobian singular (\texttt{SNES\_DIVERGED\_LINEAR\_SOLVE}, reason~$-3$,
0~iterations).  The fix constrains all three DOFs of boundary rebar
nodes:
\texttt{model.constrain\_node(rnid\_top, \{0,0,0\})}.

\paragraph{Bug fix: ghost nodes in Hex20 boundary-condition loops.}
The \texttt{PrismaticGrid::nodes\_on\_face()} helper enumerates all
grid-level indices on a face, including \emph{mid-face-centre}
nodes that do not exist in the Hex20 (serendipity) topology.
These ghost nodes carry \texttt{num\_dof() == 0} in the domain.
Passing them to \texttt{constrain\_node()}, \texttt{constrain\_dof()},
\texttt{update\_imposed\_value()}, or \texttt{extract\_base\_shear\_x()}
triggered PETSc assertion failures (invalid constraint index) or
span-index-out-of-bounds aborts.
The fix adds a guard \texttt{if (node.num\_dof() == 0) continue;}
in every face-node loop.

\paragraph{Bug fix: penalty coupling for serendipity elements.}
Two issues prevented Hex20 convergence in the penalty-coupled
column:
\begin{enumerate}
  \item \textbf{Wrong shape functions.}
    The original coupling evaluated tensor-product \emph{Hex27} shape
    functions for all quadratic meshes, including Hex20.  For
    serendipity elements, the coupling now skips face-centre and
    body-centre nodes
    ($\sum_{k}[i_k = 1] > 1$) and evaluates the standard 20-node
    serendipity shape functions:
    \[
      N_{\text{corner}} = \tfrac{1}{8}
        (1 + \xi_0\,\xi)(1 + \eta_0\,\eta)(1 + \zeta_0\,\zeta)
        (\xi_0\,\xi + \eta_0\,\eta + \zeta_0\,\zeta - 2),
    \]
    \[
      N_{\text{mid-edge}} = \tfrac{1}{4}
        (1 - t_m^2)\!\prod_{k \neq m}(1 + s_{k0}\,s_k),
    \]
    where~$m$ is the midpoint direction.
  \item \textbf{Zero-DOF rebar mid-nodes in coupling.}
    For \texttt{step\,=\,2} grids, the truss elements connect every
    other rebar node (Line2: $iz = 0 \to 2 \to 4\ldots$), leaving
    intermediate rebar nodes ($iz = 1, 3, 5, \ldots$) with
    0~DOFs.  The penalty coupling incorrectly processed these
    zero-DOF rebar nodes: the residual hook wrote penalty forces
    to wrong memory locations (since
    \texttt{PetscSectionGetOffset} returns a stale offset), while
    the Jacobian hook skipped them (guard \texttt{r\_dof\,$\leq$\,0}).
    This residual/Jacobian inconsistency produced linear
    (non-quadratic) convergence---visible as a constant 3\%
    reduction per SNES iteration---leading to divergence.
    The fix filters out zero-DOF rebar nodes during
    \texttt{PenaltyCoupling::setup()}.
\end{enumerate}

\section{Implementation Bugs Fixed}
\label{sec:cyclic-bugs}

During the validation campaign, three bugs in
\texttt{NonlinearSubModelEvolver} were discovered and fixed:

\paragraph{Bug~1: Solution vector not restored after SNES failure.}
When a SNES solve diverged in \texttt{subsequent\_solve\_adaptive()},
the solution vector~$\mathbf{U}$ was left in a corrupted state from the
failed Newton iteration.  The fix saves $\mathbf{U}$ before the SNES
call and restores it upon failure using \texttt{VecCopy}.

\paragraph{Bug~2: Imposed boundary conditions not restored.}
The \texttt{write\_imposed\_values()} call modifies the model's
\texttt{imposed\_solution} vector before each SNES solve.  Upon
SNES failure and bisection, the imposed values from the failed
sub-step were not reverted.  The fix saves and restores the imposed
solution vector with \texttt{VecCopy}, matching Bug~1.

\paragraph{Bug~3: \texttt{last\_good\_frac\_} assigned after recovery.}
The adaptive sub-stepping uses \texttt{last\_good\_frac\_} to cache the
last successful integration fraction.  After completing a full sub-step
(fraction $= 1.0$), the variable was reset to~$0$ because the
assignment occurred after the recovery branch.  This caused the next
adaptive call to restart from the beginning.  The fix moves the
assignment before the recovery line.

\paragraph{Bug~4: first-ramp continuation budget silently ignored.}
The persistent local solver stored \texttt{first\_step\_bisect\_} and exposed
\texttt{enable\_arc\_length(true)} as part of the FE\textsuperscript{2}
validation surface, but the first nonlinear solve still executed a fixed
incremental ramp with no rollback/bisection path.  As a consequence, the
driver-level contract and the actual local continuation algorithm diverged
exactly at the first cracked FE\textsuperscript{2} point.  The fix unifies the
first solve with the adaptive ramp logic, restores both the displacement and
the imposed boundary vector on failure, and enforces the minimum fraction
\(\Delta p_{\min} = 2^{-B}/N\) implied by the configured first-ramp
bisection depth \(B\) and the initial increment count \(N\).

\section{Historical Case 5 Diagnosis: Secant Tangent Limitation}
\label{sec:case5-failure}

\noindent\textit{Historical note.} This section is retained as provenance only.
The current audited interpretation is given later in
Section~\ref{sec:case5-updated-root-cause}.

The two-way FE² coupling requires the sub-model to provide a
well-conditioned homogenised tangent $\mathbf{D}_{\text{hom}}$ that
the macro-scale Newton solver uses for equilibrium iteration.
The tangent is computed by numerical perturbation:
\begin{equation}
  D_{\text{hom},ij} =
    \frac{\sigma_i(\boldsymbol{\varepsilon} + \delta\, \mathbf{e}_j)
        - \sigma_i(\boldsymbol{\varepsilon})}{\delta}
  \label{eq:dhom-perturbation}
\end{equation}
where each perturbation direction requires a separate SNES solve of
the sub-model.

After the first-solve ramp (20-increment loading from zero to the
macro-scale state), all four column sub-models developed
extensive cracking.  The degraded secant moduli $K_s, G_s$ from the
KoBathe fracture model dropped to $\sim 0.01 K_e$ (the minimum
clamp), producing an effective tangent with:
\begin{itemize}
  \item Axial stiffness $\sim 10^3$~MPa (vs.\ $\sim 4 \times 10^9$
    elastic)
  \item Bending stiffness $\leq 0$ (clamped to 1.0 by
    regularisation)
\end{itemize}

The near-zero bending stiffness caused the SNES solver to diverge for
the bending perturbation directions, producing zero columns in
$\mathbf{D}_{\text{hom}}$.  This made the macro-scale stiffness matrix
singular, causing the global Newton to fail immediately.

\paragraph{Mitigation applied.}
A diagonal clamping regularisation was added to
\texttt{compute\_homogenized\_tangent()}:
$D_{ii} = \max(D_{ii}, 1.0)$ for non-positive diagonal entries.
Additionally, a linearised force override was implemented in
\texttt{MaterialSection}:
\[
  \boldsymbol{\sigma} = \mathbf{f}_{\text{hom}}
  + \mathbf{D}_{\text{hom}}
    (\boldsymbol{\varepsilon} - \boldsymbol{\varepsilon}_{\text{ref}})
\]
which keeps the macro-scale response consistent with the tangent
during Newton iterations.

\paragraph{Fundamental limitation.}
The regularisation and linearised override help the macro solver
converge, but the underlying issue is that the sub-model tangent is
nearly rank-deficient after cracking.  This is a consequence of the
secant stiffness formulation: $K_s$ and $G_s$ decrease monotonically
with damage, producing a tangent that does not reflect the
\emph{instantaneous} stiffness change.  The solution requires a
\emph{consistent} (algorithmic) tangent, as analysed in
Sections~\ref{sec:article-crossref} and~\ref{sec:consistent-tangent}.

\section{Updated Case 5 Root Cause Audit}
\label{sec:case5-updated-root-cause}

The earlier Case~5 narrative was too narrow.  It over-attributed the first
cracked FE\textsuperscript{2} failure to the secant character of the
Ko--Bathe tangent while the implementation still contained a more basic local
continuation defect: the first nonlinear ramp of the persistent sub-model
solver ignored the configured bisection budget and also ignored the
``adaptive/arc-length from start'' flag.  In other words, the driver could ask
for an adaptive first solve, but the local solver still executed a fixed ramp
\begin{equation}
  p_k = \frac{k}{N}, \qquad k=1,\dots,N,
\end{equation}
with no rollback/bisection mechanism attached to that first solve.

\paragraph{First-ramp continuation that is now actually enforced.}
The first solve now uses the same continuation contract expected by the cyclic
validation API.  Starting from \(\Delta p_0 = 1/N\), the local ramp advances
with
\begin{equation}
  p_{n+1}^{\mathrm{trial}} = \min(p_n + \Delta p_n, 1),
\end{equation}
accepts a sub-step if the local SNES solve converges, and then expands the
next fraction according to
\begin{equation}
  \Delta p_{n+1} = \min(2 \Delta p_n, 1 - p_{n+1}),
\end{equation}
while a failed trial restores the last converged state and bisects
\begin{equation}
  \Delta p_n \leftarrow \tfrac{1}{2}\Delta p_n.
\end{equation}
With a configured first-ramp bisection depth \(B\), the minimum admissible
fraction is therefore
\begin{equation}
  \Delta p_{\min} = 2^{-B}\Delta p_0 = \frac{2^{-B}}{N}.
  \label{eq:first-ramp-min-frac}
\end{equation}

\paragraph{Strict diagnostic probe after the correction.}
With the updated solver and the strict one-step audit
\begin{verbatim}
--submodel-increments 4
--submodel-bisections 2
--submodel-adaptive-max-substeps 4
--submodel-adaptive-max-bisections 2
--submodel-snes-max-it 20
\end{verbatim}
all four sub-models now advance past the old \(k=1/4\) wall:
\begin{itemize}
  \item sub-models 0 and 2 reach \(p_{\mathrm{local}} = 0.1875\);
  \item sub-models 1 and 3 reach \(p_{\mathrm{local}} = 0.3125\).
\end{itemize}
The step still aborts, but now with the explicit failure mode
\texttt{AdaptiveMinFractionReached}.  Therefore the strict frontier is no
longer ``Newton diverged immediately at the first quarter-step''.

\paragraph{Wider one-step probe after the correction.}
The more permissive one-step audit
\begin{verbatim}
--submodel-increments 4
--submodel-bisections 2
--submodel-adaptive-max-substeps 12
--submodel-adaptive-max-bisections 4
--submodel-snes-max-it 60
\end{verbatim}
moves the frontier substantially farther.  The final
\texttt{solver\_diagnostics.csv} row now reports:
\begin{itemize}
  \item sub-model 0: \(81.25\%\) of the target, 6 accepted sub-steps,
    9 bisections, failure at \(p_{\mathrm{local}}=0.875\),
    \(\|F\| = 2.0191\);
  \item sub-model 1: \(93.75\%\) of the target, 5 accepted sub-steps,
    5 bisections, failure at \(p_{\mathrm{local}}=1.0\),
    \(\|F\| = 5.65981\);
  \item sub-model 2: \(81.25\%\) of the target, 5 accepted sub-steps,
    7 bisections, failure at \(p_{\mathrm{local}}=0.90625\),
    \(\|F\| = 2.59141\);
  \item sub-model 3: \(100\%\) of the target, 7 accepted sub-steps,
    6 bisections, converged.
\end{itemize}
The same row also shows \(72\) to \(82\) active cracked material points, up to
three cracks at a point, one no-flow stabilization pass at most, zero no-flow
destabilization events, and elastic-recovery residuals of order \(10^{-15}\).

\paragraph{Updated interpretation.}
These observations materially change the scientific diagnosis.  The missing
first-ramp continuation was a real implementation defect, and the current
Case~5 blocker is no longer best interpreted as an immediate secant-tangent
collapse.  The present frontier is the \emph{late, heavily cracked tail} of
the first micro ramp: once the local problem reaches roughly \(80\%\) to
\(100\%\) of the target drift, Newton still stalls in three of the four
columns before the minimum admissible continuation fraction is exhausted.  The
no-flow Eq.~(26b) branch and the bounded crack-status stabilization loop are
no longer the main suspects for this frontier, because the same runs report
clean no-flow diagnostics.

\paragraph{Late-tail continuation as a separate algorithmic lever.}
The persistent local solver now carries an explicit \emph{tail rescue} stage
for adaptive ramps.  If a ramp reaches a sufficiently advanced fraction
\(\hat{p}\) of its target but still aborts with
\texttt{AdaptiveMinFractionReached} or
\texttt{AdaptiveSubstepBudgetExceeded}, the remaining segment is re-parameterised
as
\begin{equation}
  \mathbf{u}(\eta)
  = \mathbf{u}(\hat{p})
    + \eta\left[\mathbf{u}(1) - \mathbf{u}(\hat{p})\right],
  \qquad \eta \in [0,1],
\end{equation}
and solved again with its own adaptive budget.  If the rescued segment starts
with a local fraction \(\Delta \eta_0\) and bisection depth \(B_{\text{tail}}\),
the minimum \emph{absolute} continuation fraction of the original target is
\begin{equation}
  \Delta p^{\text{tail}}_{\min}
  = \bigl(1-\hat{p}\bigr)\,2^{-B_{\text{tail}}}\,\Delta\eta_0.
  \label{eq:tail-rescue-min-frac}
\end{equation}
This is a standard continuation idea in the spirit of step-length control and
arc-length methods~\cite{Riks1979,Crisfield1981}: once a large part of the
target increment has already been accepted, the remaining segment should be
treated as a new continuation problem, not as an all-or-nothing failure.

\paragraph{Default \texttt{fe2\_crack50} profile after the tail-continuation pass.}
Under the current default one-step audit
\begin{verbatim}
--fe2-profile crack50
--max-steps 1
\end{verbatim}
the micro frontier moved again.  All four sub-models now reach
\(100\%\) of the first \(+2.5\)\,mm target with:
\begin{itemize}
  \item sub-model 0: 7 accepted sub-steps, 3 bisections;
  \item sub-model 1: 4 accepted sub-steps, 0 bisections;
  \item sub-model 2: 7 accepted sub-steps, 3 bisections;
  \item sub-model 3: 6 accepted sub-steps, 2 bisections.
\end{itemize}
The same \texttt{solver\_diagnostics.csv} row reports \(80\) to \(82\) active
cracked material points, up to three cracks at a point, zero no-flow
destabilization events, elastic-recovery residuals again of order
\(10^{-15}\), and zero tail-rescue activations.  In other words, the default
\texttt{fe2\_crack50} budget is now large enough that the first local cracked
ramp closes without needing the new tail rescue at all.

\paragraph{Why the frontier is now macro, not micro.}
After the four local columns converge, the step still aborts, but now with
\texttt{MacroSolveFailed}, \texttt{failed\_submodels = 0}, and explicit rollback.
The relevant constitutive object injected into the macro beam is the affine
section law
\begin{equation}
  \mathbf{s}(\mathbf{e})
  = \mathbf{s}^{(k)}_{\mu}
    + \mathbf{D}^{(k)}_{\mu}
      \left(\mathbf{e} - \mathbf{e}^{(k)}_{\mu}\right),
  \label{eq:fe2-affine-section-law}
\end{equation}
constructed from the converged micro state of fixed-point iteration \(k\).
The FE\textsuperscript{2} orchestrator now also damps the \emph{first} micro
predictor according to
\begin{equation}
  \mathbf{r}^{(0)}_{\text{pred}}
  = \omega\,\mathbf{r}^{(0)}_{\mu}
    + (1-\omega)\,\mathbf{r}^{(0)}_{\text{base}},
  \qquad
  \mathbf{r}^{(0)}_{\text{base}} = \mathbf{0},
  \label{eq:first-predictor-damping}
\end{equation}
so that the relaxation policy acts already before the second macro solve.
Even with that damping, the default Case~5 one-step audit still diverges at
the macro level.  The current scientific frontier is therefore no longer the
first cracked micro ramp itself, but the first \emph{macro} FE\textsuperscript{2}
re-solve driven by fully converged cracked-section operators.

\paragraph{Fast \texttt{fe2\_frontier\_audit} profile and observable split.}
The newer
\begin{verbatim}
--fe2-profile frontier
--max-steps 1
\end{verbatim}
profile is intentionally more restrictive.  It is not the production-like
\texttt{crack50} budget; it is a machine-readable frontier audit designed to
expose the first failed cracked point at lower runtime cost.  In the current
audited run, the uncoupled macro step converges, but the coupled step aborts
with
\begin{equation}
  \texttt{MicroSolveFailed},
  \qquad
  \texttt{failed\_submodels}=2,
  \label{eq:case5-frontier-audit-micro-failure}
\end{equation}
and the failed coupling sites are
\(\texttt{eid}=0,\texttt{gp}=0,\xi=-0.57735\) and
\(\texttt{eid}=2,\texttt{gp}=0,\xi=-0.57735\).

The new failed-step recorder semantics are important here.  Because the step
is rolled back, accepted structural quantities are undefined and are written as
\texttt{NaN}.  The micro-side constitutive observables are still recorded from
the last attempted local states.  Two different crack counts then coexist:
\begin{itemize}
  \item the open-crack observable,
    \(\left|\mathcal{G}_{\mathrm{open}}\right|\), reported through
    \texttt{total\_cracked\_gps}, \texttt{total\_cracks}, and
    \texttt{max\_opening};
  \item the crack-history observable,
    \(\left|\mathcal{G}_{\mathrm{hist}}\right|\), reported through
    \texttt{total\_active\_crack\_history\_points}.
\end{itemize}
For the current failed first point, the row now reads
\begin{equation}
  \left|\mathcal{G}_{\mathrm{open}}\right| = 0,
  \qquad
  \left|\mathcal{G}_{\mathrm{hist}}\right| = 306,
  \qquad
  \max_g n_c(g) = 3.
  \label{eq:case5-frontier-audit-observables}
\end{equation}
This means that crack history is already active at many material points even
though no open-crack observable survives the current recorder criterion.  The
validation reading is therefore sharper than before: under the cheap frontier
profile the first failed point is micro-limited and localized, but it is not
``uncracked'' in the constitutive sense.

\paragraph{Macro step-cutting as the next continuation layer.}
The next validation-oriented continuation layer now sits at that macro
frontier.  If the macro solver exposes a runtime increment size
\(\Delta p_n\) and a target-preserving \texttt{step\_to(\(\cdot\))} API, the
FE\textsuperscript{2} orchestrator may retry the same target
\(p_{n+1}^{\star} = p_n + \Delta p_n\) with reduced internal increments
\(\Delta p_n^{(k)} = \beta^k \Delta p_n\).  The key point is that the target
protocol point is preserved; only the continuation path is refined.  This is
therefore a step-cutting continuation aid, not a hidden change of cyclic
loading program.

The associated regression surface now proves that the typed FE\textsuperscript{2}
API can recover a controlled macro failure through this cutback path while
still reaching the original target time.  The full structural Case~5 frontier
remains open in the audited local harness because the rebuilt cheap probe is
still runtime-limited, but the methodological interpretation changes again:
the library now has three explicit macro-side continuation levers for the
first cracked FE\textsuperscript{2} step---predictor admissibility filtering,
operator-space macro backtracking, and target-preserving macro step cutting.
The remaining question is no longer whether the code can continue at all, but
which continuation mechanism is sufficient to close the physical Case~5 path
without overfitting the validation setup.

\paragraph{Run-management as part of the validation method.}
At the current cost scale, the full \texttt{Case~5} campaign is no longer a
throwaway command-line experiment.  The launch protocol itself is therefore
treated as part of the validation method: the repository now provides a
dedicated launcher that preserves the exact command, redirects logs, stores the
live PID, and keeps the VTK cadence explicit.  This matters scientifically
because post-failure or post-success interpretation of crack initiation and
hysteresis loops is only as trustworthy as the traceability of the long run
that produced them.

\paragraph{What remains true about the tangent audit.}
The tangent discussion is still important, but it must now be framed honestly.
The sparse condensed operator and the numerical consistent tangent remain
relevant scientific levers for robustness of the late cracked regime.
However, the present evidence does \emph{not} support the older claim that
Case~5 fails \emph{primarily} because the secant tangent immediately makes the
macro operator singular.  The first-ramp continuation defect had to be removed
before that question could even be asked cleanly.

\section{Cross-Reference: Ko--Bathe Article vs.\ Implementation}
\label{sec:article-crossref}

This section provides a detailed comparison between the constitutive
model described in Ko \& Bathe~(2026) and the
\texttt{KoBatheConcrete3D} implementation.  Each major component is
classified as \textbf{correct}, \textbf{missing}, or
\textbf{simplified}.

\subsection{Fracture Moduli (Eqs.~10--11)}

The article defines two sets of moduli:
\begin{itemize}
  \item \textbf{Secant moduli} (Eq.~10):
    $K_s = K_e / (1 + A\, r^{b-1})$, \;
    $G_s = G_e / (1 + c\, \rho^{d-1})$,
    where $r = |\sigma_o| / f'_c$ and $\rho = \tau_o / f'_c$.
  \item \textbf{Tangent (instantaneous) moduli} (Eq.~11):
    $K_c = K_e / (1 + Ab\, r^{b-1})$, \;
    $G_c = G_e / (1 + cd\, \rho^{d-1})$.
\end{itemize}

The key difference is the extra multiplicative factors $b$ and $d$ in
the denominators of the tangent moduli.  The article states:
\emph{``To obtain the constitutive matrix including cracking, also the
experimentally measured modulus is used but accounting for the
tangential (instantaneous) value instead of the secant value.''}

\textbf{Status: Correct.}  The implementation computes both sets in
\texttt{fracture\_moduli()}.  The secant moduli are used for stress
computation (correct); the tangent moduli are used in the analytical
tangent matrix (correct).

\subsection{Constitutive Matrix in Compression Coordinates (Eq.~31)}

The article defines the consistent tangent in the 3$\times$3
compression coordinate system (Eq.~31):
\begin{align}
  \mathbf{C} &= \mathbf{C}^E - \mathbf{C}^P + \mathbf{C}^C(n)
  \label{eq:tangent-decomposition} \\
  \mathbf{C}^E &= \text{isotropic}(K_s, G_s)
  \label{eq:elastic-tangent} \\
  C^{\text{ph}}_{IJ} &= \frac{\hat{e}^p_I \cdot E_I \cdot
    \hat{e}^p_J \cdot E_J}
    {\sum_k (\hat{e}^p_k)^2 E_k^2}
  \label{eq:plastic-tangent}
\end{align}
where $\hat{e}^p_i$ are normalised plastic strain directions and $E_I$
the diagonal entries of $\mathbf{C}^E$ in the principal system.

\textbf{Status: Missing.}  The current implementation returns the
tangent matrix using only the instantaneous fracture moduli $K_c, G_c$
and the cracking modification, but \textbf{does not subtract the
plastic correction $\mathbf{C}^P$}.  This means the tangent
overestimates the stiffness when plasticity is active, degrading
Newton convergence from quadratic to linear rate.  The article
explicitly states: \emph{``By using the consistent stress--strain
matrix, the solution can reach a quadratic convergence rate.''}

\subsection{Elastic Moduli Coefficients (Appendix A)}

The experimental fits from Appendix~A are:
\begin{align}
  K_e &= 11000 + 3.2\, f'_c \label{eq:Ke} \\
  G_e &= 9224 + 136\, f'_c + 3296 \times 10^{-15}\, (f'_c)^{8.273}
  \label{eq:Ge}
\end{align}
With two branches for the degradation coefficients:
\begin{equation}
  A = 0.516,\; b = 2 + 1.81 \times 10^{-8}\, (f'_c)^{4.461}
  \quad \text{(for } f'_c \leq 31.7 \text{)}
  \label{eq:A-b-low}
\end{equation}
and modified forms for $f'_c > 31.7$~MPa (Eq.~A.3).

\textbf{Status: Correct.}  All coefficients in
\texttt{KoBatheParameters} match the article.

\subsection{Plasticity (Eqs.~15, 27--29)}

The multi-surface plasticity with three independent yield functions
in compression coordinates is correctly implemented:
\begin{itemize}
  \item Hydrostatic surface ($f_1$) with power-law hardening
  \item Uniaxial compression ($f_2$) with exponential-peak hardening
  \item Biaxial compression ($f_3$) with similar form
  \item Coefficients $c_{p1} = 3$, $c_{p2} = 36$, $c_{p3} = 12$
    match Appendix~C (Eq.~C.3)
\end{itemize}

\textbf{Status: Correct} for the return mapping and stress update.
The independent surface treatment (sequential, not coupled) is
consistent with the article's decoupled approach.

\subsection{Cracking (Eqs.~17--23)}

The smeared crack model with up to 3~orthogonal cracks, tension
softening (Eq.~22), and crack closure is implemented.  The 3D
implementation uses Mandel rotation matrices for the crack-local
constitutive modification.

\textbf{Status: Correct} with a modification: the stiffness
retention factors $\eta_N = 0.20$ and $\eta_S = 0.50$ are
significantly larger than the article's $\eta_N = 10^{-4}$ and
$\eta_S = 0.1$, in order to tame the tangent discontinuity at
crack initiation in the FE² context.  A smooth sigmoid transition
(tanh) is used at the open/close boundary.

\subsection{Validation Examples}

The article presents four benchmark problems:
\begin{enumerate}
  \item Uniaxial/biaxial compression tests (Figs.~9--10)
  \item Concrete panel W-2 under shear (Fig.~11, Table~3)
  \item Scordelis--Bresler RC beams A-2, A-3, OA-2, OA-3
    (Figs.~14--21, Tables~5--8)
  \item Notched beam under bending (Figs.~22--29, Table~9)
  \item Gravity dam with initial crack (Figs.~30--34, Table~10)
\end{enumerate}

\textbf{Status:} Replication of these benchmarks is
planned but not yet performed.  The uniaxial compression test can
serve as a unit test for the material model.  The Scordelis--Bresler
beams require a full 3D mesh with embedded reinforcement (truss
elements with penalty coupling), which is already available in the
sub-model solver framework.

\subsection{Convergence Procedure}

The article emphasises the importance of the consistent tangent for
quadratic convergence.  The complete solution follows the Total
Lagrangian formulation (Eq.~24), linearised for material-nonlinearity-only
(Eq.~25):
\begin{equation}
  \int_V C_{ijrs}\, \delta e_{rs}\, de_{ij}\, dV
  = {}^{t+\Delta t}\!R
  - \int_V {}^t\!\sigma_{ij}\, \delta e_{ij}\, dV
\end{equation}
where $C_{ijrs}$ must be the \emph{algorithmic} tangent (consistent
with the return-mapping algorithm) to achieve quadratic convergence.

The load--displacement control (LDC) with line searches is used for
post-peak analysis.  The implementation's \texttt{l2} line search in
SNES is analogous to the line search described in the article
(Ref.~[36]).

\section{Consistent Tangent: Analysis and Implementation}
\label{sec:consistent-tangent}

\subsection{The Missing Plastic Tangent Contribution}

As identified in Section~\ref{sec:article-crossref}, the current
analytical tangent uses the instantaneous fracture moduli $K_c, G_c$
(Eq.~11) and applies cracking modifications (Eq.~20), but omits the
plastic return-mapping contribution $\mathbf{C}^P$ (Eq.~31c--d).

For the Ko--Bathe model with three independent yield surfaces, the
algorithmic tangent of each return-mapping surface $i$ introduces a
rank-1 correction:
\begin{equation}
  \mathbf{C}^{\text{ep}}_i = \mathbf{C}^e
  - \frac{(\mathbf{C}^e\, \mathbf{n}_i)
          (\mathbf{C}^e\, \mathbf{n}_i)^T}
         {\mathbf{n}_i^T\, \mathbf{C}^e\, \mathbf{n}_i + H_i}
  \label{eq:consistent-tangent-surface}
\end{equation}
where $\mathbf{n}_i$ is the flow direction and $H_i$ is the hardening
modulus.  Because the yield functions are defined in the compression
coordinate system and the flow directions are aligned with the
compression coordinate axes (associated flow with decoupled surfaces),
the derivation simplifies.  However, the transformation from the
3$\times$3 compression system to the 6$\times$6 global Voigt tangent
requires the chain rule through principal elastic strain decomposition,
compression coordinate mapping, and the Mandel rotation.

\subsection{Numerical Consistent Tangent}

Rather than deriving the full analytical expression, a numerical
consistent tangent was implemented using forward finite differences:
\begin{equation}
  C_{ij}^{\text{num}} =
    \frac{\sigma_i(\boldsymbol{\varepsilon} + h\, \mathbf{e}_j)
        - \sigma_i(\boldsymbol{\varepsilon})}{h}
  \label{eq:numerical-tangent}
\end{equation}
with adaptive perturbation $h = 10^{-7} \cdot
\max(|\varepsilon_j|, 10^{-6})$, followed by symmetrisation
$\mathbf{C} = \tfrac{1}{2}(\mathbf{C}^{\text{raw}} +
(\mathbf{C}^{\text{raw}})^T)$.

This approach has several advantages:
\begin{enumerate}
  \item It is \textbf{exactly consistent} with the algorithmic stress
    update, capturing all three nonlinearities (fracturing, plasticity,
    cracking) and their mutual interaction.
  \item It requires no analytical derivation of the chain rule through
    the compression coordinate and principal strain transformations.
  \item The cost (7~stress evaluations per Gauss point) is acceptable
    because the Ko--Bathe model is computationally inexpensive (no
    linear system solve in the constitutive update).
  \item The symmetrisation produces a tangent suitable for symmetric
    solvers.
\end{enumerate}

The implementation adds a \texttt{use\_consistent\_tangent\_} flag
(default: \texttt{false} for backward compatibility) and a
\texttt{set\_consistent\_tangent(bool)} setter to
\texttt{KoBatheConcrete3D}.  When enabled, the \texttt{tangent()}
methods call the numerical procedure instead of the analytical
$K_c / G_c$ tangent.

\subsection{Expected Impact on Case 5}

With the consistent tangent enabled, the sub-model SNES should
exhibit improved Newton convergence even after extensive cracking,
because:
\begin{itemize}
  \item The tangent correctly reflects the \emph{current} stiffness
    state, including the plastic return-mapping softening.
  \item The perturbation-based tangent
    (Eq.~\ref{eq:numerical-tangent}) naturally accounts for the
    crack opening/closing transitions, avoiding the discontinuity
    that causes the analytical tangent to oscillate.
  \item The improved tangent makes the numerical perturbation for
    $\mathbf{D}_{\text{hom}}$ (Eq.~\ref{eq:dhom-perturbation})
    more likely to converge, reducing the probability of zero columns.
\end{itemize}

A rerun of Case~5 with the consistent tangent is planned as future
work.

\section{FE² Allocation Audit and Optimisations}
\label{sec:fe2-alloc-audit}

An audit of the PETSc vector allocations in the FE² pipeline
(\texttt{NonlinearSubModelEvolver}) was performed.  The key findings
and subsequent optimisations:

\subsection{Audit Findings}

\begin{itemize}
  \item \texttt{FormResidual}: 2~\texttt{DMGetLocalVector} per SNES
    iteration (plus corresponding restores).
  \item \texttt{FormJacobian}: 1~\texttt{DMGetLocalVector} per SNES
    iteration.
  \item \texttt{compute\_homogenized\_tangent}: previously
    2~\texttt{VecDuplicate} per call (for saving/restoring the
    solution and imposed values).
  \item \textbf{No memory leak} was found in
    \texttt{section\_forces\_from\_reactions()} ---
    \texttt{DMRestoreLocalVector} is correctly called.
\end{itemize}

\subsection{Pre-Allocated Working Vectors (Implemented)}

The \texttt{VecDuplicate} calls in
\texttt{compute\_homogenized\_tangent()} and
\texttt{subsequent\_solve()} have been replaced by persistent
class-member vectors \texttt{U\_work\_} and \texttt{imp\_work\_},
allocated once during \texttt{setup\_snes()} and destroyed in
\texttt{destroy\_petsc\_objects()}.  This eliminates $\sim 80$
allocation/deallocation cycles per global step per sub-model
(2~allocs $\times$ 6~perturbation directions $\times$ $\sim$7
recovery calls per step).

\subsection{RCM Reordering (Implemented)}

Reverse Cuthill--McKee reordering is now applied to the direct
factoriser within each sub-model:
\begin{verbatim}
  PCFactorSetMatOrderingType(pc, MATORDERINGRCM);
  PCFactorSetReuseOrdering(pc, PETSC_TRUE);
\end{verbatim}
The reordering reduces fill-in during LU factorisation, and the
reuse flag ensures that the symbolic factorisation (which includes
the reordered sparsity pattern) is computed only once per sub-model
lifetime, with subsequent factorisation steps reusing the same
ordering.  This benefits the sub-model solver because the sparsity
pattern is constant (same mesh, same BCs) across all time steps.

\section{Implementation Notes}
\label{sec:cyclic-implementation}

The validation driver is implemented in
\texttt{main\_table\_cyclic\_validation.cpp} with a CLI dispatch:

\begin{verbatim}
  ./fall_n_table_cyclic_validation --case 0     # elastic reference
  ./fall_n_table_cyclic_validation --case 1a    # beam N=2
  ./fall_n_table_cyclic_validation --case 1i    # beam N=10
  ./fall_n_table_cyclic_validation --case 2a    # Hex8 continuum
  ./fall_n_table_cyclic_validation --case 3     # full table
  ./fall_n_table_cyclic_validation --case 4     # FE² one-way
  ./fall_n_table_cyclic_validation --case 5     # FE² two-way
  ./fall_n_table_cyclic_validation --case all   # all cases
\end{verbatim}

Each case produces a \texttt{hysteresis.csv} file (columns:
\texttt{step}, \texttt{p}, \texttt{drift\_m},
\texttt{base\_shear\_MN}) and VTK output in the subdirectory
\texttt{data/output/cyclic\_validation/case\{id\}/}.  Cases~2, 3,
4,~5 also produce PVD time-series VTK output in a \texttt{vtk/}
subdirectory.

A Python plotting script
(\texttt{scripts/plot\_cyclic\_validation.py}) reads all
\texttt{hysteresis.csv} files and generates comparative PDF figures.

\paragraph{Protocol extraction.}
The cyclic displacement function is factored into a shared header
\texttt{src/utils/CyclicProtocol.hh}, making it available for both
the validation driver and unit tests.  It is an \texttt{inline}
free function in \texttt{namespace fall\_n}, included transitively
via \texttt{header\_files.hh}.

\paragraph{Beam dispatch.}
Sub-cases \texttt{1a}--\texttt{1i} are instantiated from a single
template function \texttt{run\_case1<N>()} using a C++20 template
lambda and compile-time dispatch, avoiding code duplication across
nine element orders:
\begin{verbatim}
  auto run_beam = [&]<std::size_t N>() {
      auto records = run_case1<N>(out + "/case1" + ...);
      all_records["1" + suffix] = std::move(records);
  };
  if (should_run("1a")) run_beam.template operator()<2>();
  if (should_run("1b")) run_beam.template operator()<3>();
  // ... through 1i → N=10
\end{verbatim}

\paragraph{Key solver parameters.}
\begin{itemize}
  \item Global SNES: \texttt{bt} line search, \texttt{preonly/lu}
    KSP/PC, \texttt{max\_it}~$= 100$.
  \item Incremental solve: $N_s = 120$ steps (beam/continuum/table),
    $N_s = 60$ (FE²), max bisection depth~$= 6$.
  \item Sub-model SNES: \texttt{max\_it}~$= 100$, \texttt{atol}~$= 2.0$,
    \texttt{rtol}~$= 10^{-3}$, incremental ramp 20~sub-steps,
    4~bisection levels.
  \item Sub-model: RCM reordering + reuse, penalty
    $\alpha = 10\,E_c$, $2 \times 2 \times 1$ Hex27 mesh.
  \item Staggered coupling (Case~5): max 3~iterations,
    tolerance~$= 0.05$, relaxation~$\omega = 0.8$.
\end{itemize}
