\chapter{Normative Multiscale Publication Audit and Redesign}
\label{ch:multiscale-publication-audit}

\section{Purpose and Normative Scope}
\label{sec:ms-pub-purpose}

This chapter is the normative reference for the public multiscale API in
\texttt{fall\_n}.  It supersedes the terminology and behavioural claims of
Chapters~\ref{ch:two-way-fe2} and \ref{ch:cyclic-validation} whenever those
chapters describe an earlier or partially implemented coupling loop.

The goal of the redesign is not merely to rename components, but to align the
software architecture with three simultaneous requirements:
\begin{enumerate}
  \item \textbf{physical coherence:} the public semantics must reflect the
    actual multiscale algorithm implemented in code;
  \item \textbf{architectural extensibility:} the hot path must remain
    zero-overhead while extension points stay explicit and SOLID at the API
    boundary;
  \item \textbf{publication readiness:} code, examples, tests, and
    documentation must tell the same story.
\end{enumerate}

\section{Official Nomenclature}
\label{sec:ms-pub-nomenclature}

The multiscale subsystem now exposes three officially distinct coupling modes:
\begin{description}
  \item[\texttt{OneWayDownscaling}] macro solve followed by micro solve, with no
    feedback to the macro model;
  \item[\texttt{LaggedFeedbackCoupling}] macro solve followed by micro solve and
    delayed feedback injection for subsequent macro steps;
  \item[\texttt{IteratedTwoWayFE2}] fixed-point macro--micro iteration with
    checkpoint/restore and a single accepted commit at the end of the step.
\end{description}

This separation is implemented in
\texttt{src/analysis/CouplingStrategy.hh} and reported explicitly through
\texttt{CouplingMode} in \texttt{src/analysis/MultiscaleTypes.hh}.  The
previous labels \texttt{OneWayCoupling} and \texttt{TwoWayStaggered} are
retained only as historical names in older documentation chapters.

\section{Underlying Mathematics and Physical Contract}
\label{sec:ms-pub-math}

\subsection{Macro generalised variables}

At the beam level, the section deformation vector is
\begin{equation}
  \mathbf{e}
  =
  \{\varepsilon_0,\kappa_y,\kappa_z,\gamma_y,\gamma_z,\theta'\}^{T},
\end{equation}
and the section resultant vector is
\begin{equation}
  \mathbf{s}
  =
  \{N,M_y,M_z,V_y,V_z,T\}^{T}.
\end{equation}

The local continuum sub-model is driven by Timoshenko-consistent end
kinematics extracted from the macro beam.  The current implementation uses the
kinematic transfer already established in Chapters~\ref{ch:multiscale_seismic}
and \ref{ch:nl-sub-model-evolver}, but the public contract is no longer
``beam element to entire sub-model'' in the abstract; it is
\emph{macro coupling site to local model} through \texttt{CouplingSite}.

\subsection{Normative homogenised response}

The public response object is
\texttt{SectionHomogenizedResponse}, containing:
\begin{equation}
  \mathbf{s}_0, \qquad
  \mathbf{D}_{\mathrm{hom}} = \frac{\partial \mathbf{s}}{\partial \mathbf{e}},
  \qquad
  \mathbf{e}_0.
\end{equation}
The macro-scale injection contract is
\begin{equation}
  \mathbf{s}(\mathbf{e})
  =
  \mathbf{s}_0 + \mathbf{D}_{\mathrm{hom}}(\mathbf{e}-\mathbf{e}_0),
  \label{eq:ms-pub-linearized-response}
\end{equation}
with \(\mathbf{e}_0\) stored as \texttt{strain\_ref}.  This eliminates the
ambiguity of ``tangent by one operator, forces by another'' in the public API.
It also keeps the public contract aligned with the Hill--Mandel spirit of
energetic consistency used in computational homogenization
\cite{Hill1963,Feyel1999,Geers2010}: macro section work is represented by one
linearised operator, not by unrelated force and tangent paths.
The same response object also records the requested tangent mode
(\texttt{TangentComputationMode}), the outcome of the condensed operator
attempt (\texttt{CondensedTangentStatus}), and the residual of the sparse free
solve whenever linearized condensation is attempted.

\subsection{Boundary-reaction homogenisation}

The default production path in \texttt{NonlinearSubModelEvolver} now computes
section forces from assembled boundary reactions at the loaded end.  In compact
form,
\begin{equation}
  \mathbf{s}_{\mathrm{hom}}
  =
  \mathcal{H}_{\mathrm{br}}(\mathbf{u}_{\mu}),
\end{equation}
where \(\mathcal{H}_{\mathrm{br}}\) denotes the boundary-reaction operator and
\(\mathbf{u}_{\mu}\) is the converged micro displacement field.  The
publication-path tangent is obtained primarily from the linearised micro
equilibrium equations.  With the free/constrained partition
\(\{f,c\}\) and constrained increments
\(\delta \mathbf{u}_c = \mathbf{T}\,\delta \mathbf{q}\), the partitioned micro
equilibrium problem is
\begin{equation}
  \begin{bmatrix}
    \mathbf{K}_{ff} & \mathbf{K}_{fc} \\
    \mathbf{K}_{cf} & \mathbf{K}_{cc}
  \end{bmatrix}
  \begin{Bmatrix}
    \delta \mathbf{u}_f \\
    \delta \mathbf{u}_c
  \end{Bmatrix}
  =
  \begin{Bmatrix}
    \mathbf{0} \\
    \delta \mathbf{r}_c
  \end{Bmatrix},
  \label{eq:ms-pub-partitioned-micro-equilibrium}
\end{equation}
so that the free increment is
\begin{equation}
  \delta \mathbf{u}_f
  =
  -\mathbf{K}_{ff}^{-1}\mathbf{K}_{fc}\mathbf{T}\,\delta \mathbf{q},
  \label{eq:ms-pub-free-increment}
\end{equation}
and the constrained reaction increment becomes
\begin{equation}
  \delta \mathbf{r}_c
  =
  \left(
    \mathbf{K}_{cc}
    -
    \mathbf{K}_{cf}\mathbf{K}_{ff}^{-1}\mathbf{K}_{fc}
  \right)\mathbf{T}\,\delta \mathbf{q}.
  \label{eq:ms-pub-constrained-reaction}
\end{equation}
Projecting those reactions back to the section basis yields the condensed
section operator
\begin{equation}
  \mathbf{D}_{\mathrm{hom}}
  =
  \mathbf{T}^{\mathsf T}
  \left(
    \mathbf{K}_{cc}
    -
    \mathbf{K}_{cf}\mathbf{K}_{ff}^{-1}\mathbf{K}_{fc}
  \right)\mathbf{T}.
  \label{eq:ms-pub-condensed-section-operator}
\end{equation}
This is the static-condensation identity that underlies Guyan reduction and
later component-mode substructuring
\cite{Guyan1965,CraigBampton1968}.  In matrix-analysis terms, the elimination
of the interior block is the Schur complement of \(\mathbf{K}_{ff}\)
\cite{Zhang2005Schur}.  The implementation evaluates
\eqref{eq:ms-pub-condensed-section-operator} through a sparse solve of the six
right-hand sides in \(\mathbf{K}_{fc}\mathbf{T}\), not by materialising a full
dense Schur complement, and reuses the symbolic sparsity pattern whenever the
micro topology is unchanged in the usual sparse-direct sense
\cite{Davis2006Sparse}.

Adaptive finite differences remain available as an explicit validation and
fallback path.  The perturbation length for generalized component \(j\) is
chosen as
\begin{equation}
  h_j
  =
  \frac{\max\!\left(h_{\mathrm{pert}}L,\;
    10^{-3}\max(|q_j|, h_{\mathrm{pert}}L)\right)}{L},
  \label{eq:ms-pub-adaptive-step}
\end{equation}
where \(L\) is the beam characteristic length and \(q_j\) is the corresponding
relative end degree of freedom.  This is not claimed to be an optimal
finite-difference step rule; it is a pragmatic scale-aware heuristic inspired
by the step-size balance arguments discussed by
Gill et al.~\cite{GillMurraySaundersWright1983}.  The preferred tangent
estimate is then
\begin{equation}
  \mathbf{D}_{\mathrm{hom}}^{(:,j)}
  \approx
  \frac{\mathcal{H}_{\mathrm{br}}(\mathbf{u}_{\mu}^{(j,+)})
        - \mathcal{H}_{\mathrm{br}}(\mathbf{u}_{\mu}^{(j,-)})}{2h_j},
  \label{eq:ms-pub-central-fd}
\end{equation}
with one-sided fallback only when one perturbation fails.  The public API can
force this route through
\texttt{TangentComputationMode::ForceAdaptiveFiniteDifference} for validation.

The runtime audit mode
\texttt{ValidateCondensationAgainstAdaptiveFiniteDifference} compares the
condensed operator against the adaptive finite-difference reference.  The
legacy metric retained for exact reproducibility is the relative Frobenius gap
\begin{equation}
  \eta_{F}
  =
  \frac{\lVert \mathbf{D}_{\mathrm{cond}}
         - \mathbf{D}_{\mathrm{fd}} \rVert_F}
       {\max\!\left(1,
           \lVert \mathbf{D}_{\mathrm{cond}} \rVert_F,
           \lVert \mathbf{D}_{\mathrm{fd}} \rVert_F\right)},
  \label{eq:ms-pub-validation-gap}
\end{equation}
with per-column diagnostics
\begin{equation}
  \eta_j
  =
  \frac{\lVert \mathbf{D}_{\mathrm{cond}}^{(:,j)}
         - \mathbf{D}_{\mathrm{fd}}^{(:,j)} \rVert_2}
       {\max\!\left(1,
           \lVert \mathbf{D}_{\mathrm{cond}}^{(:,j)} \rVert_2,
           \lVert \mathbf{D}_{\mathrm{fd}}^{(:,j)} \rVert_2\right)}.
  \label{eq:ms-pub-validation-column-gap}
\end{equation}

The limitation of \(\eta_F\) is not algebraic but physical: the six columns of
\(\mathbf{D}_{\mathrm{hom}}\) do not represent equally relevant directions at a
given converged state, and the raw matrix entries mix force and moment outputs.
Consequently, a pure Frobenius norm may reject a condensed tangent because of
errors in columns that are nearly dormant in the current macro state, even when
the operator action along the actually active directions is already accurate
enough for the next FE\textsuperscript{2} fixed-point iteration.  This is the
main scientific limitation of \(\eta_F\): it is algebraically legitimate, but
it is blind to the current work-conjugate state of the section.

To address that gap, the current implementation adds three weighted
alternatives.
The default publication-path metric is
\texttt{TangentValidationNormKind::StateWeightedFrobenius}.  Let
\((b,h)\) denote the local rectangular section dimensions and define the
half-span scales
\begin{equation}
  c_y = \frac{h}{2}, \qquad
  c_z = \frac{b}{2}, \qquad
  c_t = \frac{1}{2}\sqrt{b^2+h^2}.
\end{equation}
With the current generalized strain proxy
\(\mathbf{e}_{\mathrm{ref}} =
\{\varepsilon_0,\kappa_y,\kappa_z,\gamma_y,\gamma_z,\theta'\}^{T}\)
extracted from the same beam-to-continuum kinematics used by the local solve,
the state amplitudes are
\begin{equation}
  \widehat{\mathbf{e}}_{\mathrm{ref}}
  =
  \left\{
    |\varepsilon_0|,
    c_y |\kappa_y|,
    c_z |\kappa_z|,
    |\gamma_y|,
    |\gamma_z|,
    c_t |\theta'|
  \right\}^{T},
  \label{eq:ms-pub-state-weighted-amplitudes}
\end{equation}
and the column weights are
\begin{equation}
  w_j
  =
  \max\!\left(
    \frac{\widehat e_{\mathrm{ref},j}}
         {\max_k \widehat e_{\mathrm{ref},k}},
    10^{-3}
  \right),
  \qquad
  \mathbf{W}_q = \mathrm{diag}(w_1,\dots,w_6).
  \label{eq:ms-pub-state-weighted-columns}
\end{equation}
The runtime validation gap then becomes
\begin{equation}
  \eta_{\mathrm{sw}}
  =
  \frac{\lVert
      (\mathbf{D}_{\mathrm{cond}}-\mathbf{D}_{\mathrm{fd}})\mathbf{W}_q
    \rVert_F}
       {\max\!\left(
          1,
          \lVert \mathbf{D}_{\mathrm{cond}}\mathbf{W}_q \rVert_F,
          \lVert \mathbf{D}_{\mathrm{fd}}\mathbf{W}_q \rVert_F
       \right)}.
  \label{eq:ms-pub-state-weighted-gap}
\end{equation}
This is the norm currently used by default because it measures the error of the
operator action along the directions that are actually active in the current
increment.  That makes it a materially better proxy for FE\textsuperscript{2}
fixed-point convergence than a raw all-columns-equal matrix norm.

The next option,
\texttt{TangentValidationNormKind::DualEnergyScaled}, augments the same
activity-weighted column operator \(\mathbf{W}_q\) with a row scaling extracted
from the current generalized force state
\(\mathbf{s}_{\mathrm{ref}}=
\{N,M_y,M_z,V_y,V_z,T\}^{T}\).  Define
\begin{equation}
  \widehat{\mathbf{s}}_{\mathrm{ref}}
  =
  \left\{
    |N|, |M_y|, |M_z|, |V_y|, |V_z|, |T|
  \right\}^{T},
\end{equation}
the state floor
\(\epsilon_e = 10^{-3}\),
the force floor
\(\epsilon_s = 10^{-3}\max_i \widehat s_{\mathrm{ref},i}\),
and the reference power scale
\begin{equation}
  P_{\mathrm{ref}}
  =
  \sum_{i=1}^{6}
  \max(\widehat s_{\mathrm{ref},i}, \epsilon_s)\,
  \max(\widehat e_{\mathrm{ref},i}, \epsilon_e).
  \label{eq:ms-pub-dual-energy-scale}
\end{equation}
The dual row operator is then
\begin{equation}
  \mathbf{W}_d
  =
  \mathrm{diag}(d_1,\dots,d_6),
  \qquad
  d_i
  =
  \frac{\max_j \widehat s_{\mathrm{ref},j}\,
        \max(\widehat e_{\mathrm{ref},i}, \epsilon_e)}
       {P_{\mathrm{ref}}}.
  \label{eq:ms-pub-dual-energy-rows}
\end{equation}
The corresponding runtime validation metric becomes
\begin{equation}
  \eta_{\mathrm{de}}
  =
  \frac{\lVert
      \mathbf{W}_d(\mathbf{D}_{\mathrm{cond}}-\mathbf{D}_{\mathrm{fd}})
      \mathbf{W}_q
    \rVert_F}
       {\max\!\left(
          1,
          \lVert \mathbf{W}_d \mathbf{D}_{\mathrm{cond}}\mathbf{W}_q \rVert_F,
          \lVert \mathbf{W}_d \mathbf{D}_{\mathrm{fd}}\mathbf{W}_q \rVert_F
       \right)}.
  \label{eq:ms-pub-dual-energy-gap}
\end{equation}
This metric is not presented as a proof of exact energetic equivalence.  Its
purpose is more modest and more practical: it penalizes discrepancies that are
simultaneously aligned with the currently active generalized strains and the
currently active generalized forces, making it a better proxy for whether the
accepted operator will materially change the next generalized work increment.

The second weighted option,
\texttt{TangentValidationNormKind::SectionPowerScaledFrobenius}, is a more
conservative audit metric.  It keeps the same state-weighted column operator
\(\mathbf{W}_q\) but also scales the output resultants into
force-equivalent units through
\begin{equation}
  \mathbf{W}_s
  =
  \mathrm{diag}\!\left(
    1,\frac{1}{c_y},\frac{1}{c_z},1,1,\frac{1}{c_t}
  \right),
  \label{eq:ms-pub-section-power-row-scaling}
\end{equation}
which yields
\begin{equation}
  \eta_{\mathrm{sp}}
  =
  \frac{\lVert
      \mathbf{W}_s(\mathbf{D}_{\mathrm{cond}}-\mathbf{D}_{\mathrm{fd}})
      \mathbf{W}_q
    \rVert_F}
       {\max\!\left(
          1,
          \lVert \mathbf{W}_s \mathbf{D}_{\mathrm{cond}}\mathbf{W}_q \rVert_F,
          \lVert \mathbf{W}_s \mathbf{D}_{\mathrm{fd}}\mathbf{W}_q \rVert_F
       \right)}.
  \label{eq:ms-pub-section-power-gap}
\end{equation}
This norm is useful when the audit must emphasise work-conjugate
force-versus-moment distinctions explicitly.  In practice it is more
conservative than \(\eta_{\mathrm{sw}}\) and therefore remains available as an
auditing option rather than as the default production gate.

The API now records the selected norm kind, the row scales, and the column
weights in every \texttt{SectionHomogenizedResponse}.  The automatic
accept/reject decision uses whichever metric is requested by
\texttt{TangentValidationNormKind}
(\texttt{RelativeFrobenius}, \texttt{StateWeightedFrobenius},
\texttt{DualEnergyScaled}, or \texttt{SectionPowerScaledFrobenius});
\(\max_j \eta_j\) remains visible as a diagnostic rather than a hard gate
because per-column maxima are still more useful as an interpretive tool than
as a standalone production criterion.

Volume averaging is retained only as a comparison/validation route and is
reported through \texttt{HomogenizationOperator::VolumeAverage}; it is not the
default FE\textsuperscript{2} production operator.

\subsection{Convergence criterion}

The implemented FE\textsuperscript{2} loop now uses the same norm family for
outer fixed-point residuals that is already used to validate the condensed
operator.  For coupling site \(i\), let
\(\mathbf{e}_{\mathrm{ref}}^{(i)}\) denote the current macro generalized-strain
state extracted at that site and let \(\mathbf{W}_q^{(i)}\) and
\(\mathbf{W}_s^{(i)}\) be the state-weighting and force-equivalent row-scaling
operators defined in \eqref{eq:ms-pub-state-weighted-columns} and
\eqref{eq:ms-pub-section-power-row-scaling}.  The legacy raw force residual is
\begin{equation}
  r_{f,F}^{(i)}
  =
  \frac{\lVert \mathbf{s}_{\text{macro}}^{(i)}
  - \mathbf{s}_{\text{micro}}^{(i)} \rVert_2}
  {\max\!\left(1,
    \lVert \mathbf{s}_{\text{macro}}^{(i)} \rVert_2,
    \lVert \mathbf{s}_{\text{micro}}^{(i)} \rVert_2\right)},
  \label{eq:ms-pub-force-residual-raw}
\end{equation}
while the legacy raw tangent residual between two consecutive FE\textsuperscript{2}
iterates, \(\mathbf{D}^{(i,k)}\) and \(\mathbf{D}^{(i,k-1)}\), is
\begin{equation}
  r_{D,F}^{(i,k)}
  =
  \frac{\lVert \mathbf{D}^{(i,k)} - \mathbf{D}^{(i,k-1)} \rVert_F}
  {\max\!\left(1,
    \lVert \mathbf{D}^{(i,k)} \rVert_F,
    \lVert \mathbf{D}^{(i,k-1)} \rVert_F\right)}.
  \label{eq:ms-pub-tangent-residual-raw}
\end{equation}

The state-weighted force residual now used on the publication path is
\begin{equation}
  r_{f,\mathrm{sw}}^{(i)}
  =
  \frac{\lVert \mathbf{W}_q^{(i)}
    (\mathbf{s}_{\text{macro}}^{(i)}-\mathbf{s}_{\text{micro}}^{(i)})
  \rVert_2}
  {\max\!\left(1,
    \lVert \mathbf{W}_q^{(i)}\mathbf{s}_{\text{macro}}^{(i)} \rVert_2,
    \lVert \mathbf{W}_q^{(i)}\mathbf{s}_{\text{micro}}^{(i)} \rVert_2\right)},
  \label{eq:ms-pub-force-residual-state-weighted}
\end{equation}
and the corresponding state-weighted tangent residual is
\begin{equation}
  r_{D,\mathrm{sw}}^{(i,k)}
  =
  \frac{\lVert
    (\mathbf{D}^{(i,k)}-\mathbf{D}^{(i,k-1)})\mathbf{W}_q^{(i)}
  \rVert_F}
  {\max\!\left(1,
    \lVert \mathbf{D}^{(i,k)}\mathbf{W}_q^{(i)} \rVert_F,
    \lVert \mathbf{D}^{(i,k-1)}\mathbf{W}_q^{(i)} \rVert_F\right)}.
  \label{eq:ms-pub-tangent-residual-state-weighted}
\end{equation}
The dual-energy alternative available in the same family is
\begin{equation}
  r_{f,\mathrm{de}}^{(i)}
  =
  \frac{\lVert \mathbf{W}_d^{(i)}
    (\mathbf{s}_{\text{macro}}^{(i)}-\mathbf{s}_{\text{micro}}^{(i)})
  \rVert_2}
  {\max\!\left(1,
    \lVert \mathbf{W}_d^{(i)}\mathbf{s}_{\text{macro}}^{(i)} \rVert_2,
    \lVert \mathbf{W}_d^{(i)}\mathbf{s}_{\text{micro}}^{(i)} \rVert_2\right)},
  \label{eq:ms-pub-force-residual-dual-energy}
\end{equation}
and
\begin{equation}
  r_{D,\mathrm{de}}^{(i,k)}
  =
  \frac{\lVert
    \mathbf{W}_d^{(i)}(\mathbf{D}^{(i,k)}-\mathbf{D}^{(i,k-1)})
    \mathbf{W}_q^{(i)}
  \rVert_F}
  {\max\!\left(1,
    \lVert \mathbf{W}_d^{(i)}\mathbf{D}^{(i,k)}\mathbf{W}_q^{(i)} \rVert_F,
    \lVert \mathbf{W}_d^{(i)}\mathbf{D}^{(i,k-1)}\mathbf{W}_q^{(i)}
    \rVert_F\right)}.
  \label{eq:ms-pub-tangent-residual-dual-energy}
\end{equation}
Finally, the conservative section-power-scaled family available for audit
campaigns is
\begin{equation}
  r_{f,\mathrm{sp}}^{(i)}
  =
  \frac{\lVert \mathbf{W}_s^{(i)}\mathbf{W}_q^{(i)}
    (\mathbf{s}_{\text{macro}}^{(i)}-\mathbf{s}_{\text{micro}}^{(i)})
  \rVert_2}
  {\max\!\left(1,
    \lVert \mathbf{W}_s^{(i)}\mathbf{W}_q^{(i)}
      \mathbf{s}_{\text{macro}}^{(i)} \rVert_2,
    \lVert \mathbf{W}_s^{(i)}\mathbf{W}_q^{(i)}
      \mathbf{s}_{\text{micro}}^{(i)} \rVert_2\right)},
  \label{eq:ms-pub-force-residual-section-power}
\end{equation}
and
\begin{equation}
  r_{D,\mathrm{sp}}^{(i,k)}
  =
  \frac{\lVert
    \mathbf{W}_s^{(i)}(\mathbf{D}^{(i,k)}-\mathbf{D}^{(i,k-1)})
    \mathbf{W}_q^{(i)}
  \rVert_F}
  {\max\!\left(1,
    \lVert \mathbf{W}_s^{(i)}\mathbf{D}^{(i,k)}\mathbf{W}_q^{(i)} \rVert_F,
    \lVert \mathbf{W}_s^{(i)}\mathbf{D}^{(i,k-1)}\mathbf{W}_q^{(i)}
    \rVert_F\right)}.
  \label{eq:ms-pub-tangent-residual-section-power}
\end{equation}

The important architectural consequence is that
\texttt{ForceAndTangentConvergence} remains a small tolerance policy, but the
meaning of the residuals it consumes is now configurable and physically
observable through \texttt{CouplingIterationReport}.  The default
\texttt{MultiscaleAnalysis} configuration uses
\texttt{StateWeightedFrobenius} for both force and tangent residuals, while the
report records the chosen norm family together with the effective component or
row/column scales applied at each site.  The runtime-benchmarked
\texttt{DualEnergyScaled} alternative remains available for campaigns where the
acceptance criterion should track the current generalized work content more
closely than a strain-only weighting.

\section{Implemented Architecture}
\label{sec:ms-pub-architecture}

\subsection{Public data contracts}

The redesign introduces explicit public types:
\begin{itemize}
  \item \texttt{CouplingSite}: identifies the exact macro site
    (\texttt{macro\_element\_id}, section Gauss point, \(\xi\), local frame);
  \item \texttt{MacroSectionState}: local macro strain/resultant snapshot;
  \item \texttt{SectionHomogenizedResponse}: homogenised forces, tangent,
    reference strain, operator provenance, regularisation diagnostics,
    linearisation provenance, condensed-attempt diagnostics,
    tangent-column validity, and perturbation sizes when the
    finite-difference fallback is actually used;
  \item \texttt{CouplingIterationReport}: mode, termination reason,
    iteration count, norm-aware force/tangent residuals, the residual norm
    kinds and effective residual scales, hard-failure sites, rollback state,
    regularisation presence, and macro/micro timing;
  \item \texttt{ModelCheckpoint} and solver-specific
    \texttt{checkpoint\_type}: exact rollback state for FE\textsuperscript{2}
    iteration.
\end{itemize}

The typed path through \texttt{MultiscaleModel<MacroBridgeT, LocalModelT>} is
the normative public surface.  The type-erased
\texttt{LocalModelHandle} remains available only as an experimental utility
until erased checkpoint/restore semantics are completed.  Likewise,
\texttt{src/reconstruction/HomogenisationStrategy.hh} is retained as an
internal validation utility rather than a recommended public extension point.

\subsection{Material and local-method extensibility boundary}

At the FE\textsuperscript{2} orchestration level, the local problem should be
understood as a history-dependent operator rather than as ``the Ko--Bathe
continuum solve''.  The mathematically relevant object is
\begin{equation}
  \mathcal{L}_{\mu} :
  \left(
    \mathbf{e}_{n+1}^{\mathrm{trial}},
    \mathbf{z}_{n},
    t_n,
    \Delta t
  \right)
  \mapsto
  \left(
    \mathbf{s}_{n+1},
    \mathbf{D}_{n+1},
    \mathbf{z}_{n+1},
    \mathcal{D}_{n+1}
  \right),
  \label{eq:ms-pub-local-operator-contract}
\end{equation}
where \(\mathbf{e}_{n+1}^{\mathrm{trial}}\) is the generalized strain state
imposed by the macro problem, \(\mathbf{z}_n\) is the local internal history,
\(\mathbf{s}_{n+1}\) is the homogenized section response,
\(\mathbf{D}_{n+1}\) is the linearized section operator used by the outer
iteration, and \(\mathcal{D}_{n+1}\) is the diagnostic payload.  Nothing in
\eqref{eq:ms-pub-local-operator-contract} requires \(\mathbf{z}_n\) to come
from a particular constitutive family.  It may encode continuum crack history,
enriched degrees of freedom, discontinuous traces, or any other local state
representation.

The implementation now reflects that separation explicitly.  The protocol owned
by \texttt{MultiscaleAnalysis} depends only on the
\texttt{LocalModelAdapter} concept, while the persistent continuum local model
(\texttt{NonlinearSubModelEvolver}) obtains its material instances through the
contracts \texttt{ConcreteMaterialFactory} and
\texttt{RebarMaterialFactory}, defined in
\texttt{src/materials/SubmodelMaterialFactory.hh}.  The current reference
implementations
(\texttt{KoBatheConcreteMaterialFactory} and
\texttt{MenegottoPintoRebarFactory}) now live in
\texttt{src/materials/SubmodelMaterialFactoryDefaults.hh}, and
\texttt{src/reconstruction/MaterialFactory.hh} is reduced to a compatibility
shim rather than a normative ownership point.

This architectural boundary is essential for scientific extensibility.  Future
local models may be built as enriched crack solvers in the extended finite
element sense \cite{BelytschkoBlack1999}, or as discontinuous
Petrov--Galerkin / broken-test-space local solves in the sense of
Demkowicz and Gopalakrishnan \cite{DemkowiczGopalakrishnan2010}, without
rewriting the FE\textsuperscript{2} orchestration layer.  The multiscale kernel
therefore depends on the local operator contract
\eqref{eq:ms-pub-local-operator-contract}, not on the accidental fact that the
current reference local operator happens to be a Ko--Bathe-driven reinforced
continuum solve.

\subsection{SOLID without sacrificing the hot path}

The multiscale orchestration now follows the design principle ``static in the
kernel, explicit at the boundary'':
\begin{itemize}
  \item \textbf{Single Responsibility Principle.}
    \texttt{BeamMacroBridge} is responsible for macro extraction/injection;
    \texttt{MultiscaleModel} aggregates coupling sites and local models;
    \texttt{MultiscaleAnalysis} owns the protocol; the local solver owns local
    constitutive evolution.  Internally, the nonlinear sub-model path now
    isolates boundary-condition application, persistent state lifecycle, and
    boundary-reaction homogenisation into dedicated header-only components.
  \item \textbf{Open/Closed Principle.}
    Coupling algorithms, convergence criteria, relaxation policies, and
    executors are all pluggable through small interfaces.
  \item \textbf{Liskov Substitution Principle.}
    Macro solvers participate through the
    \texttt{CheckpointableSteppableSolver} concept rather than through ad hoc
    assumptions about \texttt{DynamicAnalysis} or \texttt{NonlinearAnalysis}.
  \item \textbf{Interface Segregation Principle.}
    The local-model concept requires only what the multiscale loop actually
    needs: kinematics update, solve, homogenised response, commit/revert, and
    checkpoint control.
  \item \textbf{Dependency Inversion Principle.}
    \texttt{MultiscaleAnalysis} depends on abstract coupling policies and on
    concepts, not on concrete drivers.  The same rule now applies one level
    lower: the multiscale and reconstruction layers depend on material-factory
    contracts owned by \texttt{src/materials}, while Ko--Bathe/Menegotto remain
    replaceable reference implementations rather than hard-coded assumptions of
    the FE\textsuperscript{2} core.
\end{itemize}

These abstractions are intentionally placed at the orchestration layer.  The
inner loop remains template-based and avoids virtual dispatch in the hot path.
This preserves the original performance philosophy of \texttt{fall\_n} while
making the multiscale subsystem externally reusable.

At the local-solver level, the same principle is now applied internally:
\begin{itemize}
  \item \texttt{LocalBoundaryConditionApplicator} owns beam-to-continuum
    kinematic updates and writes the imposed boundary vector;
  \item \texttt{PersistentLocalStateOps} owns synchronisation,
    commit, and revert of the persistent continuum state;
  \item \texttt{BoundaryReactionHomogenizer} owns the section-force extraction,
    tangent diagnostics, linearized static condensation, finite-difference
    fallback, and response classification;
  \item \texttt{LocalCrackDiagnostics} owns the reduction from constitutive
    fracture fields to the stable public \texttt{CrackSummary} and, when
    requested, the detailed \texttt{CrackRecord} geometry;
  \item \texttt{LocalVTKOutputWriter} owns PVD/VTK emission so that the local
    solver lifecycle and the local visualization lifecycle are no longer
    entangled.
\end{itemize}
These components are value-type, header-only helpers with pointer-based
composition; they add no virtual dispatch to the micro hot path while creating
an explicit seam for future replacement of the current sparse condensation /
validation-fallback pair by even more advanced local linearisation
strategies.

This cycle also extended \texttt{FEM\_Element} to mirror the explicit local
linearisation API already present in \texttt{StructuralElement}
(\texttt{extract\_element\_dofs}, \texttt{compute\_internal\_force\_vector},
\texttt{compute\_tangent\_stiffness\_matrix}, and
\texttt{get\_dof\_indices}).  That architectural change matters because the
publication-ready local operator now depends on mixed-model assembly through
type erasure without losing access to the element-level tangent information.

\subsection{Condensed operator implementation}

The current primary local operator is no longer assembled as a dense
micro-tangent followed by dense Schur complements.  Instead, the implementation
builds a sparse local tangent from the mixed element set, adds the embedded
penalty coupling contributions, and solves only the six right-hand sides needed
to evaluate
\begin{equation}
  \mathbf{D}_{\mathrm{hom}}
  =
  \mathbf{T}^{\mathsf T}
  \left(
    \mathbf{K}_{cc}
    -
    \mathbf{K}_{cf}\mathbf{K}_{ff}^{-1}\mathbf{K}_{fc}
  \right)\mathbf{T}.
\end{equation}
Operationally, this is implemented as a sparse solve for
\(\mathbf{X} = \mathbf{K}_{ff}^{-1}\mathbf{K}_{fc}\mathbf{T}\), followed by
the boundary contraction
\(\mathbf{T}^{\mathsf T}(\mathbf{K}_{cc}\mathbf{T}-\mathbf{K}_{cf}\mathbf{X})\).
This keeps the physics of static condensation while reducing memory pressure
and making the scalability story substantially more honest than the previous
dense prototype.  The current stabilized implementation goes one step further:
the sparse Schur solve is now factored into a reusable workspace that caches
the symbolic factorisation pattern of \(\mathbf{K}_{ff}\) across repeated local
operator evaluations.  That distinction matters for HPC honesty because it
separates topology-dependent symbolic work from state-dependent numerical
factorisation.

\subsection{Validation and fallback diagnostics}

The current public contract does not hide whether the condensed operator was
actually available.  Instead, the response records:
\begin{itemize}
  \item the requested tangent mode;
  \item the realised tangent scheme;
  \item the condensed operator status;
  \item the runtime validation status of the condensed operator against the
    adaptive finite-difference reference;
  \item the validation tolerance together with the measured global and
    per-column gaps;
  \item the sparse solve residual when the condensed path is attempted;
  \item whether the symbolic factorisation pattern was reused;
  \item how many symbolic pattern analyses have been performed by the active
    sparse Schur workspace.
\end{itemize}
The intended semantics are:
\begin{description}
  \item[\texttt{PreferLinearizedCondensation}] production mode; use the sparse
    condensed operator first and fall back only if it cannot be formed;
  \item[\texttt{ValidateCondensationAgainstAdaptiveFiniteDifference}] audit
    mode; compute the condensed operator, compare it against an adaptive
    finite-difference reference, evaluate the selected validation norm
    (\texttt{RelativeFrobenius}, \texttt{StateWeightedFrobenius},
    \texttt{DualEnergyScaled}, or
    \texttt{SectionPowerScaledFrobenius}), accept the condensed operator if the
    measured gap stays below the configured tolerance, otherwise fall back
    explicitly to the finite-difference tangent;
  \item[\texttt{ForceAdaptiveFiniteDifference}] validation mode; skip
    condensation intentionally and compute the tangent through adaptive
    perturbations;
  \item[\texttt{TangentValidationNormKind}] explicit operator metric family:
    the legacy raw Frobenius norm for exact reproducibility,
    the default state-weighted norm for production gating,
    and the more conservative section-power-scaled norm for auditing;
  \item[\texttt{TangentValidationStatus}] explicit result of the runtime audit:
    \texttt{Accepted}, \texttt{Rejected}, or
    \texttt{ReferenceUnavailable};
  \item[\texttt{CondensedTangentStatus}] explicit reason code for the
    condensed attempt, including \texttt{Success},
    \texttt{AssemblyFailed}, \texttt{FactorizationFailed},
    \texttt{ResidualTooLarge}, \texttt{ValidationRejected}, and
    \texttt{ForcedAdaptiveFiniteDifference}.
\end{description}
This additional provenance is important for publication because it makes the
scientific debt visible in the API instead of burying it inside solver logs.

\subsection{Macro and local rollback}

The FE\textsuperscript{2} loop is only meaningful if the macro and micro
states can both be restored exactly.  The current implementation provides:
\begin{itemize}
  \item model-level snapshots through \texttt{ModelCheckpoint};
  \item solver-level snapshots in \texttt{DynamicAnalysis} and
    \texttt{NonlinearAnalysis};
  \item local-model snapshots in \texttt{NonlinearSubModelEvolver};
  \item explicit trial/commit control via \texttt{set\_auto\_commit(false)}
    during iteration and a single \texttt{commit\_trial\_state()} after
    acceptance.
\end{itemize}

\section{Implemented Iterated Two-Way FE\textsuperscript{2} Algorithm}
\label{sec:ms-pub-algorithm}

Algorithm~\ref{alg:iterated-two-way-fe2} summarises the implemented
orchestrator semantics.

\begin{algorithm}[htb]
\DontPrintSemicolon
\caption{Implemented \texttt{IteratedTwoWayFE2} protocol}
\label{alg:iterated-two-way-fe2}
\KwIn{macro solver \(S_M\), multiscale model \(\mathcal{M}\),
      predictor response \(\mathbf{R}^{(0)}\)}
\KwOut{accepted macro step and committed local states}
\BlankLine
capture macro checkpoint \(C_M\)\;
capture one checkpoint \(C_\mu^{(i)}\) for each local model\;
disable automatic macro and local commit\;
\For{\(k = 0,1,\dots,k_{\max}-1\)}{
  restore \(C_M\) and all \(C_\mu^{(i)}\)\;
  inject predictor response at each \texttt{CouplingSite}\;
  advance one macro step\;
  extract local beam kinematics for each site\;
  solve local models in parallel\;
  build \texttt{SectionHomogenizedResponse} for each site\;
  compute force and tangent residuals\;
  apply relaxation policy if configured\;
  \If{\(k \ge 1\) and all convergence criteria are satisfied}{
    accept current iteration\;
    \textbf{break}\;
  }
  set predictor \(\leftarrow\) current response\;
}
inject accepted response at each site\;
commit macro trial state\;
commit local trial states and call \texttt{end\_of\_step()} once\;
\end{algorithm}

Two implementation decisions are deliberate:
\begin{enumerate}
  \item the first iteration is always treated as a predictor pass and is never
    accepted immediately;
  \item \texttt{end\_of\_step()} is owned by the orchestrator, not by the
    driver.
\end{enumerate}

\section{HPC and Build Surface}
\label{sec:ms-pub-hpc}

The first publication-ready HPC layer focuses on robust shared-memory
parallelism:
\begin{itemize}
  \item \texttt{SerialExecutor} provides the reference path;
  \item \texttt{OpenMPExecutor} parallelises local-model solves over coupling
    sites;
  \item \texttt{CouplingCommunicators} makes the future MPI ownership explicit
    without pretending that a distributed FE\textsuperscript{2} engine is
    already complete.
\end{itemize}

The build system now exposes a dedicated public CMake target:
\begin{center}
\texttt{fall\_n::multiscale\_api}
\end{center}
and a slim public umbrella header:
\begin{center}
\texttt{src/analysis/MultiscaleAPI.hh}
\end{center}
which gathers the multiscale-specific headers without routing users through the
project-wide \texttt{header\_files.hh} umbrella.

\section{Known Limitations}
\label{sec:ms-pub-limitations}

The redesign closes the most serious semantic gaps, but several publication
limitations remain explicit:
\begin{itemize}
  \item the default section tangent is now obtained by linearized static
    condensation of the assembled local tangent operator
    \(\mathbf{D}_{\mathrm{hom}} = \mathbf{T}^{\mathsf T}
    \mathbf{K}_{\mathrm{eff}} \mathbf{T}\), but the implementation is still a
    dense local condensation and falls back to adaptive finite differences when
    mixed-element local linearisation is unavailable or the free block becomes
    numerically singular;
  \item the accepted FE\textsuperscript{2} step is a fixed-point iteration, not
    a monolithic multilevel Newton solve;
  \item MPI-distributed local-model execution is not yet implemented beyond the
    communicator contract;
  \item the default validation and FE\textsuperscript{2} residual norms are now
    state-weighted, which is a much better convergence proxy than raw
    Frobenius in the current beam-driven setting, but they are still built from
    the generalized strain coordinates of the present transfer operator rather
    than from a fully systematic dual energy norm over the microproblem;
  \item the section-power-scaled norm is intentionally conservative and may
    reject operators that the state-weighted norm accepts; this is a feature
    for audit campaigns, not a proof that the condensed operator is
    inadequate for the current outer iteration;
  \item per-column validation gaps remain diagnostic only; the present code does
    not yet expose a second independent acceptance tolerance for
    \(\max_j \eta_j\);
  \item \texttt{NonlinearSubModelEvolver} still contains several responsibilities
    internally, but the boundary-condition, state-lifecycle, and
    homogenisation concerns have already been split from the main class body;
  \item the GitHub Actions workflow added for multiscale stability has been
    authored and documented, but only the local PowerShell gate has been
    executed in this audit cycle;
  \item historical chapters still contain earlier design narratives and should
    be read as provenance, not as the normative behavioural specification.
\end{itemize}

\section{Future Work}
\label{sec:ms-pub-future}

The next publication-strength milestones are:
\begin{enumerate}
  \item generalise the current condensed local tangent into a fully systematic
    algorithmic linearisation path that covers every local hook and minimises
    the need for the finite-difference fallback while retaining the same
    public response contract;
  \item replace the present generalized-strain proxy weights by a genuinely
    dual work norm derived from the local incremental variational problem, so
    that the already aligned validation and FE\textsuperscript{2} convergence
    paths use a truly energetic metric rather than the present kinematics-based
    proxy;
  \item finish the internal split of \texttt{NonlinearSubModelEvolver} from
    the current state into a solver core, diagnostics service, and output
    service, reusing the already extracted boundary-condition, state, and
    homogenisation helpers;
  \item add energy-consistency benchmarks and distributed MPI execution of
    local models;
  \item package installable public headers for the multiscale target instead of
    exposing the source-tree include path;
  \item extend the coupling-site abstraction beyond beam sections to shell or
    continuum macro sites.
\end{enumerate}

\section{Conclusion}
\label{sec:ms-pub-conclusion}

The multiscale subsystem of \texttt{fall\_n} is now documented and exposed as a
library feature rather than as a driver convention.  The redesign does not
claim that every research-grade FE\textsuperscript{2} feature is complete; it
does claim, and the current code supports, that:
\begin{enumerate}
  \item the coupling modes are semantically distinct and named honestly;
  \item iterated two-way FE\textsuperscript{2} performs real macro/micro
    checkpointed iteration;
  \item forces, tangents, and reference strain are injected through one public
    response object;
  \item the lifecycle no longer leaks into the examples;
  \item the documentation now states these facts explicitly.
\end{enumerate}
